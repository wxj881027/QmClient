# 脚本工具调研报告

## 概述

本报告对 QmClient 项目中 `scripts/` 和 `qmclient_scripts/` 两个目录下的所有脚本工具进行了全面调研，梳理了每个脚本的功能、用途、依赖关系和适用场景。

---

## 一、`scripts/` 目录 — 上游构建与代码质量工具

该目录主要继承自 DDNet 上游项目，包含构建辅助、代码质量检查、代码生成、国际化翻译工具和依赖库编译脚本。

### 1.1 构建辅助脚本

| 脚本 | 语言 | 功能说明 |
|------|------|----------|
| `cmake-windows.cmd` | Batch | Windows 构建 wrapper，自动查找并加载 VS2022 的 VsDevCmd 环境（x64），将 Ninja 加入 PATH，然后透传所有参数给 cmake。是 Windows 上构建项目的核心入口 |
| `android/cmake_android.sh` | Bash | Android 平台构建脚本，支持 arm/arm64/x86/x86_64 四种架构，使用 Android NDK 交叉编译，配置 Gradle 项目结构，生成 APK 所需的 .so 库和资源文件 |

### 1.2 代码质量检查脚本

| 脚本 | 语言 | 功能说明 |
|------|------|----------|
| `check_config_variables.py` | Python | 检查 `config_variables.h` 中定义的配置变量是否在源码中被实际使用。解析 `MACRO_CONFIG_*` 宏定义，然后在 `src/` 下搜索 `g_Config.m_*` 等引用，报告未使用的配置变量 |
| `check_dilate.py` | Python | 检查 PNG 图片是否已经过 dilate（膨胀）处理。递归扫描目录下的 .png 文件，调用 `dilate --dry-run` 验证，用于确保游戏皮肤纹理的边缘处理正确 |
| `check_header_guards.py` | Python | 检查头文件的 include guard 是否符合命名规范。根据文件路径生成期望的 guard 名称（如 `src/engine/client/foo.h` → `ENGINE_CLIENT_FOO_H`），报告不符合规范的文件 |
| `check_identifiers.py` | Python | 配合 `extract_identifiers.py` 使用，从 CSV 输入中检查标识符命名是否符合 Teeworlds 匈牙利命名法（m_ 前缀、C/I 类名前缀、大驼峰等） |
| `check_standard_headers.py` | Python | 检查 C++ 源码中是否使用了 C 风格的标准库头文件（如 `<stdio.h>`），建议替换为 C++ 风格（如 `<cstdio>`）。上游 CI `style.yml` 中使用 |
| `check_unused_header_files.py` | Python | 检查 `src/` 下是否有未被任何源文件 include 的头文件 |
| `fix_style.py` | Python | 自动格式化代码风格。查找 clang-format 20，对 `src/` 下的 C/C++ 文件执行格式化，同时确保文件末尾有换行符。支持 `--dry-run` 模式仅检查不修改 |
| `extract_identifiers.py` | Python | 使用 libclang 解析 C++ 源文件，提取标识符信息（变量名、类名、函数名等），输出 CSV 格式。是 `check_identifiers.py` 的上游工具 |

### 1.3 代码生成脚本

| 脚本 | 语言 | 功能说明 |
|------|------|----------|
| `gen_keys.py` | Python | 从 `SDL_scancode.h` 生成键码定义文件。生成三个文件：`src/engine/keys.h`（EKey 枚举）、`src/engine/client/keynames.cpp`（键名字符串数组）、`src/engine/client/keynames.h`（声明）。包含键盘、鼠标、摇杆的键码映射 |
| `generate_unicode_confusables_data.py` | Python | 从 Unicode 官方数据生成易混淆字符分解表。读取 `UnicodeData.txt` 和 `confusables.txt`，生成 `src/base/unicode/confusables.h` 和 `confusables_data.h`，用于用户名反欺诈检测 |
| `generate_unicode_tolower.py` | Python | 从 Unicode 官方数据生成大写到小写映射表。读取 `UnicodeData.txt`，生成 `src/base/unicode/tolower_data.h` 和 `tolower_data.cpp`，用于不区分大小写的文本比较 |
| `unicode.py` | Python | Unicode 数据解析辅助模块，提供 `confusables()`、`data()`、`unhex()` 等函数，供上述两个脚本共用 |
| `wordlist.py` | Python | 从 `data/wordlist.txt` 生成 C++ 头文件 `g_aFallbackWordlist` 数组，用于游戏内随机名称生成等功能 |
| `checksum.py` | Python | 计算源文件和额外数据的 SHA256 校验和，生成 `CChecksumData::InitFiles()` 函数代码，用于客户端完整性校验 |
| `git_revision.py` | Python | 获取当前 Git 提交的短哈希值（优先从 `DDNET_GIT_SHORTREV_HASH` 环境变量），生成 `GIT_SHORTREV_HASH` 常量定义，嵌入到构建产物中 |
| `export_settings_commands_table.py` | Python | 解析 `config_variables.h` 和源码中的 `Register()` 调用，导出所有配置变量和控制台命令的 HTML 表格文档，按类别（服务端/客户端/Econ/聊天）分组输出 |

### 1.4 服务器管理脚本

| 脚本 | 语言 | 功能说明 |
|------|------|----------|
| `hash_passwords.py` | Python | 将明文密码转换为 DDNet 配置文件可用的哈希格式。解析 `auth_add` 命令行，生成 `auth_add_p` 命令（含 MD5 哈希 + 随机盐值），支持批量处理配置文件 |
| `import_file_score.py` | Python | 将旧版文本格式的成绩记录（`*_record.dtb`）导入 SQLite 数据库。解析玩家名、完成时间、25 个检查点时间，写入 `record_race` 表 |
| `move_sqlite.py` | Python | 定期迁移 SQLite 数据库中的成绩记录。将活跃数据库中的记录转移到归档数据库，支持备份表、时间戳时区转换，设计用于 cron 定时任务 |
| `generate_community_token.py` | Python | 生成社区令牌（community token），用于主服务器（mastersrv）认证。生成 user_token 和 mastersrv_token，包含 CRC32 校验和 SHA256 哈希 |
| `generate_fake_curl.py` | Python | 生成 libcurl 的 stub 共享库（.so），用于链接测试。扫描源码中所有 `curl_*` 函数调用，创建对应的空函数符号 |

### 1.5 测试脚本

| 脚本 | 语言 | 功能说明 |
|------|------|----------|
| `integration_test.py` | Python | 集成测试框架，启动客户端、服务端和主服务器进程进行端到端测试。包含完整的进程管理、日志解析、超时控制、FIFO 命令管道。内置测试用例：服务端启动、客户端启动、客户端连接、编辑器打开、冒烟测试、主服务器注册等 |

### 1.6 平台特定脚本

| 脚本 | 语言 | 功能说明 |
|------|------|----------|
| `darwin_fix_install_names.py` | Python | macOS 专用。修复 App Bundle 中动态库的 install_name，将绝对路径改为 `@rpath/` 相对路径，确保 dylib 在 Frameworks 目录下能正确加载 |
| `darwin_strip_rpath.py` | Python | macOS 专用。从可执行文件中删除所有 LC_RPATH 加载命令，用于发布前清理构建路径信息 |
| `send_named_pipe.ps1` | PowerShell | Windows 专用。连接到命名管道服务器发送消息，用于与运行中的客户端/服务端通过 FIFO 管道通信 |
| `parse_crashlog_drmingw.py` | Python | 解析 DrMingw 生成的崩溃日志，支持 lzma 压缩日志，使用 argparse 命令行参数，将地址转换为源码位置，辅助 Windows 崩溃调试 |

### 1.7 网络工具脚本

| 脚本 | 语言 | 功能说明 |
|------|------|----------|
| `tw_api.py` | Python | Teeworlds 主服务器 API 客户端。连接 4 个官方主服务器获取服务器列表，再逐个查询服务器信息（版本、地图、玩家数等），统计在线玩家和观众数量 |

### 1.8 代码生成与构建辅助脚本（恢复自上游）

| 脚本 | 语言 | 功能说明 |
|------|------|----------|
| `generate_rust_bridge.py` | Python | 生成 Rust FFI 桥接代码，用于 C++ 与 Rust 模块的互操作。项目包含少量 Rust 模块时必需 |
| `compile_libs/_build_common.sh` | Bash | 依赖库编译的公共函数脚本，被其他 `make_lib_*.sh` 脚本引用 |
| `android/download_android_sdk.sh` | Bash | 自动下载 Android SDK，上游 CI `build.yml` 中使用 |
| `android/generate_asset_integrity_index.py` | Python | 生成 Android 资源完整性索引文件，用于 APK 资源校验 |

### 1.9 国际化翻译工具（`scripts/languages/`）

| 脚本 | 语言 | 功能说明 |
|------|------|----------|
| `twlang.py` | Python | DDNet 翻译文件解析核心模块。提供 `decode()` 解析翻译文件格式、`check_folder()` 扫描源码中的 `Localize/Localizable` 调用、`languages()` 获取所有语言列表、`translations()` 读取翻译条目 |
| `analyze.py` | Python | 分析各语言翻译完成度。统计每个语言文件的条目总数、空翻译数、缺失条目数、未使用条目数，以表格形式输出 |
| `validate.py` | Python | 验证翻译质量。检查 C 格式字符串占位符是否匹配（如 `%d`、`%s`），检查省略号使用（`…` vs `...`） |
| `copy_fix.py` | Python | 翻译文件修复工具。支持删除未使用条目（`--delete-unused`）、追加缺失条目（`--append-missing`）、删除空翻译（`--delete-empty`） |
| `update_all.py` | Python | 批量更新所有 DDNet 语言翻译文件，执行 copy_fix（删除未用 + 追加缺失） |
| `find_unchanged.py` | Python | 查找翻译文件中原文与译文完全相同的条目（即未翻译的条目） |
| `from_csv.py` | Python | 将 Weblate 导出的 CSV 文件转换回 DDNet 翻译文件格式 |
| `to_csv.py` | Python | 将 DDNet 翻译文件导出为 Weblate 兼容的 CSV 格式，用于在线翻译协作 |
| **`qmclient/twlang_qmclient.py`** | Python | QmClient 专用翻译模块，基于 `twlang.py` 修改。搜索 `TCLocalize/TCLocalizable` 宏（而非 `Localize/Localizable`），读取 `data/qmclient/languages/` 下的翻译文件 |
| **`qmclient/copy_fix_qmclient.py`** | Python | QmClient 专用翻译修复工具，功能与 `copy_fix.py` 相同但使用 `twlang_qmclient` 模块 |
| **`qmclient/update_all_qmclient.py`** | Python | QmClient 专用批量翻译更新，对所有 QmClient 语言文件执行 copy_fix |

### 1.10 依赖库编译脚本（`scripts/compile_libs/`）

| 脚本 | 语言 | 功能说明 |
|------|------|----------|
| `gen_libs.sh` | Bash | 依赖库编译总入口。自动 clone 并编译 OpenSSL、zlib、libpng、curl、freetype2、SDL2、ogg、opus、opusfile、sqlite3，支持 Android/Linux/Windows/WebAssembly 目标平台，编译完成后将产物复制到 `ddnet-libs/` 目录 |
| `cmake_lib_compile.sh` | Bash | CMake 方式编译依赖库的通用脚本，被 `gen_libs.sh` 调用。支持 Android NDK 和 Emscripten (WebAssembly) 两种交叉编译方式 |
| `make_lib_openssl.sh` | Bash | OpenSSL 编译脚本，支持 Android（arm/arm64/x86/x86_64）和 WebAssembly，使用 OpenSSL 自带的 Configure + make 构建系统 |
| `make_lib_opusfile.sh` | Bash | opusfile 编译脚本，手动编译 opusfile.c/info.c/stream.c/internal.c 为静态库，依赖预先编译的 libogg 和 libopus |
| `make_lib_sqlite3.sh` | Bash | SQLite3 编译脚本，直接编译 sqlite3.c 为静态库，启用多线程检查和原子写入特性 |

## 二、`qmclient_scripts/` 目录 — QmClient 专属脚本

该目录包含 QmClient 项目特有的运维、构建和开发辅助脚本。所有非官方脚本已从 `scripts/` 迁移至此，确保 `scripts/` 与上游 DDNet 保持一致。

### 2.1 构建辅助

| 脚本 | 语言 | 功能说明 |
|------|------|----------|
| `cmake-windows.cmd` | Batch | Windows 构建 wrapper，自动查找并加载 VS2022 的 VsDevCmd 环境（x64），将 Ninja 加入 PATH，然后透传所有参数给 cmake。**非官方脚本**，从 `scripts/` 迁移至此 |
| `darwin_fix_install_names.py` | Python | macOS 专用。修复 App Bundle 中 dylib 的 install_name，将绝对路径改为 `@rpath/` 相对路径。**非官方脚本**，为修复 WebP 库 dylib 依赖问题而添加，被 `CMakeLists.txt` 的 `package_default` target 调用 |
| `make_lib_openssl.sh` | Bash | OpenSSL 编译脚本，支持 Android 和 WebAssembly。**非官方脚本**，官方 `compile_libs/` 中无此文件 |

### 2.2 客户端自动更新系统

| 脚本 | 语言 | 功能说明 |
|------|------|----------|
| `diff_update.py` | Python | **版本差异计算器**。比较两个版本的 QmClient 发布包（Win64 + Linux x86_64），递归计算文件 SHA256 哈希，生成增量更新描述（新增/修改/删除的文件列表），输出到 `update.json`。这是客户端自动更新功能的核心组件 |
| `update.zsh` | Zsh | **自动更新发布流程**。完整的版本发布自动化脚本：1) 从 GitHub Releases 下载新旧版本压缩包；2) 解压并调用 `diff_update.py` 计算差异；3) 将更新文件、可执行文件、data 目录部署到更新服务器；4) 生成 `info.json` 版本信息。设计用于 Linux 服务器上的定时/手动发布 |

### 2.3 中心识别服务器

| 脚本 | 语言 | 功能说明 |
|------|------|----------|
| `qmclient_center_server/server.js` | Node.js | **QmClient 中心识别服务器**。基于 Express 框架，提供三个 API 接口：`GET /token`（获取认证令牌）、`POST /report`（上报在线玩家信息）、`GET /users.json`（查询在线 QmClient 用户）。支持 HMAC-SHA256 令牌认证、IP 绑定、速率限制、自动过期清理。用于客户端识别其他 QmClient 用户（显示特殊标记/尾迹效果） |
| `qmclient_center_server/package.json` | JSON | Node.js 项目配置，依赖 Express 4.21.2 |
| `qmclient_center_server/README.md` | Markdown | 中心服务器使用文档，包含启动方式、API 契约、环境变量配置、生产部署建议 |

### 4.2 QmClient 专属脚本的价值

1. **自动更新系统**（`diff_update.py` + `update.zsh`）：实现了客户端增量更新能力，用户无需每次下载完整包，只需下载变更的文件。这是 QmClient 相对于原版 DDNet 的重要运维优势。

2. **中心识别服务器**（`qmclient_center_server/`）：提供了 QmClient 用户在线识别能力，使客户端能在游戏中标记其他 QmClient 用户。这是社区归属感的重要功能。

3. **QmClient 翻译工具**（`languages/qmclient/`）：独立于上游的翻译管理流程，支持 QmClient 特有 UI 文本的本地化。

### 4.3 潜在改进点

1. **`update.zsh` 的 GitHub URL**：脚本中硬编码了 `sjrc6/TaterClient-ddnet` 的 GitHub Releases URL，应确认是否需要更新为当前 QmClient 仓库地址。

2. **中心服务器存储**：当前使用内存存储（Map），生产环境建议迁移到 Redis，以支持多实例部署和持久化。

3. **`diff_update.py` 的平台覆盖**：当前仅支持 Win64 和 Linux x86_64 两个平台的差异计算，如需支持 macOS 或更多架构需扩展。

4. **脚本依赖管理**：部分 Python 脚本依赖 `clang.cindex`（libclang）、`tqdm` 等第三方库，但项目未提供 `requirements.txt`，建议补充。

## 五、上游脚本适配指南

QmClient 作为 DDNet/TaterClient 的 Fork 项目，拥有大量自有修改。上游 `scripts/` 中的部分脚本在 QmClient 环境下无法直接使用，需要适配后放入 `qmclient_scripts/`。以下是系统性分析。

### 5.1 适配必要性分析

| 上游脚本 | 是否需要适配 | 原因 |
|----------|:---:|------|
| `check_config_variables.py` | **是** | 上游仅扫描 `config_variables.h`，QmClient 有 3 个配置文件（含 `Tc`/`Qm`/`RiVoice` 前缀） |
| `check_header_guards.py` | **是** | QmClient 头文件保护宏混用 `TCLIENT`/`QMCLIENT`/`QM_UI`/`RCLIENT` 四种前缀 |
| `export_settings_commands_table.py` | **是** | 需要解析 3 个配置文件，并区分 QmClient 专属设置（语音/客户端） |
| `fix_style.py` | **可选** | 上游版本可直接使用，但 QmClient 版增加了 `--qm-only` 参数便于局部格式化 |
| `gen_keys.py` | 否 | 直接操作 SDL 头文件，与 QmClient 修改无关 |
| `generate_unicode_*.py` | 否 | 直接操作 Unicode 数据，与 QmClient 修改无关 |
| `wordlist.py` | 否 | 直接操作词表文件，与 QmClient 修改无关 |
| `checksum.py` | 否 | 由 CMake 调用，文件列表由构建系统生成 |
| `git_revision.py` | 否 | 由 CMake 调用，环境变量名可保持一致 |
| `integration_test.py` | 否 | 二进制文件名未变，但 QmClient 特有功能暂无测试用例 |
| `hash_passwords.py` | 否 | 服务端工具，与客户端修改无关 |
| `check_dilate.py` | 否 | 检查 PNG 图片，与代码修改无关 |
| `check_standard_headers.py` | 否 | 检查 C 风格头文件，与 QmClient 修改无关 |
| `check_unused_header_files.py` | 否 | 检查未使用头文件，通用工具 |
| `check_identifiers.py` | 否 | 检查命名规范，通用工具 |
| `languages/` 翻译工具 | **已有** | `qmclient/` 子目录已有适配版本 |

### 5.2 QmClient 适配脚本详解

#### `qmclient_scripts/check_config_variables.py`

**与上游的区别：**

| 特性 | 上游 | QmClient 适配版 |
|------|------|-----------------|
| 配置文件 | 仅 `config_variables.h` | 3 个文件：`config_variables.h` + `config_variables_qmclient.h` + `config_variables_qimeng.h` |
| 变量前缀 | 无特殊前缀 | `Tc`（TaterClient）、`Qm`（QmClient）、`RiVoice`（语音模块） |
| 过滤模式 | 无 | `--ddnet` 仅检查上游、`--qm` 仅检查 QmClient 自有 |

**运行结果（首次扫描发现 28 个未使用配置变量）：**

```
[qmclient] 未使用的 Tc 前缀变量 23 个（侧栏、瀑布、跳跃提示、UI 显示等）
[qimeng]   未使用的 Qm/RiVoice 前缀变量 5 个（语音指示器、饼菜单、复读键等）
```

这些变量可能是在开发中定义但尚未接入，或是已废弃但未清理。建议逐一确认后处理。

#### `qmclient_scripts/check_header_guards.py`

**与上游的区别：**

| 特性 | 上游 | QmClient 适配版 |
|------|------|-----------------|
| 保护宏规则 | 严格按路径生成 | 支持 `GUARD_OVERRIDES` 映射表 |
| QmClient 特殊命名 | 不识别 | 识别 `TCLIENT`/`QMCLIENT`/`QM_UI`/`RCLIENT` 四种前缀 |
| 过滤模式 | 无 | `--qm-only` 仅检查 QmClient 目录 |

**QmClient 头文件保护宏命名现状：**

| 目录 | 前缀 | 示例 | 来源 |
|------|------|------|------|
| `qmclient/*.h`（大部分） | `TCLIENT` | `GAME_CLIENT_COMPONENTS_TCLIENT_TCLIENT_H` | 继承自 TaterClient |
| `qmclient/*.h`（新增） | `QMCLIENT` | `GAME_CLIENT_COMPONENTS_QMCLIENT_JELLY_TEE_H` | QmClient 自研 |
| `qmclient/voice_core.h` | `RCLIENT` | `GAME_CLIENT_COMPONENTS_RCLIENT_VOICE_H` | 继承自 RiClient |
| `QmUi/*.h` | `QM_UI` | `GAME_CLIENT_QM_UI_QM_RT_H` | QmClient UI 框架 |

> **建议**：新增文件统一使用 `QMCLIENT` 前缀，旧文件保持不变以减少改动风险。

#### `qmclient_scripts/export_settings_commands_table.py`

**与上游的区别：**

| 特性 | 上游 | QmClient 适配版 |
|------|------|-----------------|
| 配置文件源 | 仅 `config_variables.h` | 3 个配置文件合并解析 |
| 输出分类 | 服务端/客户端/Econ/聊天 | 额外增加 QmClient 客户端设置、QmClient 语音设置 |
| 过滤模式 | 无 | `--qm-only` 仅导出 QmClient 专属设置 |

**QmClient 配置域架构：**

```
ConfigDomain::DDNET   → config_variables.h          → settings_ddnet.cfg
ConfigDomain::QIMENG  → config_variables_qmclient.h → settings_qmclient.cfg
ConfigDomain::QIMENG  → config_variables_qimeng.h   → settings_qmclient.cfg
```

#### `qmclient_scripts/fix_style.py`

**与上游的区别：**

| 特性 | 上游 | QmClient 适配版 |
|------|------|-----------------|
| 忽略目录 | `generated`、`rust-bridge` | 同上游 |
| 过滤模式 | 无 | `--qm-only` 仅格式化 QmClient 目录 |

### 5.3 上游脚本直接使用指南

以下上游脚本无需适配，可直接在 QmClient 项目根目录运行：

```powershell
# 代码格式化（推荐日常使用）
python scripts/fix_style.py --dry-run     # 检查格式
python scripts/fix_style.py               # 自动修复

# 配置变量检查（仅检查上游部分，QmClient 部分用适配版）
python scripts/check_config_variables.py  # 仅检查 config_variables.h

# 头文件保护宏检查（上游文件）
python scripts/check_header_guards.py     # 会报 QmClient 文件为"错误"

# 代码质量检查
python scripts/check_standard_headers.py  # 检查 C 风格头文件
python scripts/check_dilate.py data/      # 检查皮肤图片

# 翻译工具
python scripts/languages/update_all.py                    # 更新 DDNet 翻译
python scripts/languages/qmclient/update_all_qmclient.py  # 更新 QmClient 翻译
```

### 5.4 QmClient 适配脚本使用指南

```powershell
# 配置变量检查
python qmclient_scripts/check_config_variables.py          # 检查所有 3 个配置文件
python qmclient_scripts/check_config_variables.py --qm     # 仅检查 QmClient 自有配置
python qmclient_scripts/check_config_variables.py --ddnet  # 仅检查上游配置

# 头文件保护宏检查
python qmclient_scripts/check_header_guards.py             # 检查所有头文件
python qmclient_scripts/check_header_guards.py --qm-only   # 仅检查 QmClient 目录
python qmclient_scripts/check_header_guards.py --fix       # 输出修复建议

# 设置文档导出
python qmclient_scripts/export_settings_commands_table.py > docs/settings.html
python qmclient_scripts/export_settings_commands_table.py --qm-only > docs/qm_settings.html

# 代码格式化
python qmclient_scripts/fix_style.py --dry-run --qm-only   # 仅检查 QmClient 文件
python qmclient_scripts/fix_style.py --qm-only             # 仅格式化 QmClient 文件
```
