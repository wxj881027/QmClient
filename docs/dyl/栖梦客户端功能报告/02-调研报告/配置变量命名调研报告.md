# QmClient 配置变量命名调研报告

## 一、背景

QmClient 的代码继承链为：**DDNet 官方 → TClient (TaterClient) → QmClient (栖梦客户端)**。每一层都引入了自己的配置变量，但当前命名前缀和文件归属存在混乱，需要梳理和规范化。

---

## 二、配置文件结构

| 文件 | 变量数量 | 归属层 | 期望前缀 |
|---|---|---|---|
| `config_variables.h` | ~644 | DDNet 官方 | `cl_`、`gfx_`、`snd_` 等 |
| `config_variables_qmclient.h` | ~236 | TClient (上游) | `tc_` |
| `config_variables_qimeng.h` | ~194 | QmClient (本客户端) | `qm_` |

## 三、当前前缀使用统计

| `snd_` | 音频 | `snd_volume`, `snd_enable` |
| `inp_` | 输入 | `inp_mousesens` |

> ✅ DDNet 的前缀体系是清晰的，按功能域划分，无需调整。

### 3.3 `config_variables_qimeng.h`（QmClient 本客户端）

**前缀混乱**，存在 4 种不同前缀：

| 前缀 | C++ 前缀 | 数量 | 用途 | 是否规范 |
|---|---|---|---|---|
| `qm_` | `Qm` | ~145 | QmClient 通用功能 | ✅ 正确 |
| `qmc_` | `Qmc` | ~1 | 粒子特效 | ⚠️ 应为 `qm_` |
| `ri_` | `Ri` | ~48 | 语音聊天（RiVoice） | ⚠️ 应为 `qm_voice_` |
| `cl_` | `Cl` | ~2 | 计分板 | ❌ 与 DDNet 上游 `cl_` 冲突 |

## 四、问题详细分析

### 问题 1：`ri_` 前缀（RiVoice 语音模块）

**现状**：语音聊天模块使用 `ri_` 前缀（C++ 成员 `Ri`），但脚本名（ScriptName）实际使用的是 `qm_voice_*`。

```cpp
// C++ 成员名前缀是 Ri，但脚本名是 qm_voice_
MACRO_CONFIG_INT(RiVoiceEnable, qm_voice_enable, 0, ...)
MACRO_CONFIG_INT(RiVoiceProtocolVersion, qm_voice_protocol_version, 3, ...)
```

**问题**：
- C++ 成员名 `RiVoiceEnable` 与脚本名 `qm_voice_enable` 不一致
- `Ri` 前缀来源不明（可能是原始模块名 "RiVoice" 的缩写）
- 用户在控制台输入的是 `qm_voice_*`，但代码中引用的是 `g_Config.m_RiVoiceEnable`

**建议**：统一为 `qm_voice_` / `QmVoice` 前缀，或保留 `ri_` / `Ri` 但将脚本名也改为 `ri_voice_*` 保持一致。

### 问题 2：`qmc_` 前缀（粒子特效）

**现状**：

```cpp
MACRO_CONFIG_INT(QmcFootParticles, qmc_foot_particles, 0, ...)
```

**问题**：`qmc_` 与 `qm_` 含义相同（都是 QmClient），但前缀不同，造成混淆。

**建议**：改为 `qm_foot_particles` / `QmFootParticles`。

### 问题 3：`cl_` 前缀冲突（计分板）

**现状**：

```cpp
MACRO_CONFIG_INT(ClScoreboardPoints, cl_scoreboard_points, 0, ...)
MACRO_CONFIG_INT(ClScoreboardSortMode, cl_scoreboard_sort_mode, 0, ...)
```

**问题**：
- `cl_` 是 DDNet 上游的前缀，QmClient 使用 `cl_` 会与上游变量混淆
- 这两个变量是 QmClient 新增的，不是 DDNet 上游的
- 用户无法从变量名判断其归属

**建议**：改为 `qm_scoreboard_points` / `QmScoreboardPoints`。

### 问题 4：翻译功能配置归属

**现状**：翻译功能的 8 个配置变量全部使用 `tc_` 前缀，存放在 `config_variables_qmclient.h`（TClient 文件）中。

**分析**：
- 翻译功能最初由 TClient 实现，使用 `tc_` 前缀是合理的
- QmClient 新增的翻译功能（如智谱AI后端、翻译按钮、出站翻译）属于 QmClient 自身功能
- 新增配置应使用 `qm_` 前缀，存放在 `config_variables_qimeng.h`

**建议**：
- 保留现有 `tc_translate_*` 变量不动（上游代码，不应修改）
- 新增的 QmClient 翻译配置使用 `qm_translate_*` 前缀

### 问题 5：聊天气泡配置重复

**现状**：`config_variables_qmclient.h` 中有"旧版"聊天气泡配置（`tc_` 前缀），`config_variables_qimeng.h` 中有"新版"聊天气泡配置（`qm_` 前缀）：

| 旧版 (tc_) | 新版 (qm_) | 说明 |
|---|---|---|
| `tc_hide_chat_bubbles` | `qm_hide_chat_bubbles` | 隐藏聊天气泡 |
| `tc_chat_bubble` | `qm_chat_bubble` | 显示聊天气泡 |
| `tc_chat_bubble_duration` | `qm_chat_bubble_duration` | 显示时长 |
| `tc_chat_bubble_alpha` | `qm_chat_bubble_alpha` | 透明度 |
| `tc_chat_bubble_font_size` | `qm_chat_bubble_font_size` | 字体大小 |
| `tc_chat_bubble_bg_color` | `qm_chat_bubble_bg_color` | 背景色 |
| `tc_chat_bubble_text_color` | `qm_chat_bubble_text_color` | 文字色 |
| `tc_chat_bubble_animation` | `qm_chat_bubble_animation` | 动画 |

**问题**：功能重复，用户配置时可能混淆使用哪一套。

**建议**：QmClient 版本已完全替代旧版，应标记旧版为废弃（Legacy），并在代码中引导使用新版。

## 五、前缀规范建议

### 规范定义

| 前缀 | C++ 成员前缀 | 归属 | 文件 | 说明 |
|---|---|---|---|---|
| `cl_` / `gfx_` / `snd_` 等 | `Cl` / `Gfx` / `Snd` 等 | DDNet 官方 | `config_variables.h` | 按功能域划分，不修改 |
| `tc_` | `Tc` | TClient 上游 | `config_variables_qmclient.h` | TClient 的所有配置 |
| `qm_` | `Qm` | QmClient 本客户端 | `config_variables_qimeng.h` | QmClient 的所有配置 |

### 新增配置变量规则

1. **QmClient 新增功能**必须使用 `qm_` 前缀，存放在 `config_variables_qimeng.h`
2. **不得使用 `tc_` 前缀**，该前缀专属于 TClient 上游
3. **不得使用 `cl_` 等上游前缀**，避免与 DDNet 官方变量混淆
4. **子模块可使用二级前缀**，如 `qm_voice_*`、`qm_translate_*`、`qm_pie_menu_*`
5. **C++ 成员名前缀必须与脚本名前缀一致**，如 `qm_voice_enable` → `QmVoiceEnable`

### 迁移优先级

| 优先级 | 修改项 | 影响范围 | 风险 |
|---|---|---|---|
| 🔴 高 | `qmc_foot_particles` → `qm_foot_particles` | 1 个变量，低 | 低 |
| 🔴 高 | `cl_scoreboard_*` → `qm_scoreboard_*` | 2 个变量 | 中（与上游同名） |
| 🟡 中 | `Ri*` / `ri_*` → `QmVoice*` / `qm_voice_*` | ~48 个变量 | 中（数量多） |
| 🟢 低 | 旧版聊天气泡 `tc_chat_bubble_*` 标记废弃 | 8 个变量 | 低（仅标记） |

## 六、配置文件归属原则

### 判断流程

```
新增配置变量
    │
    ├─→ 该功能是否由 TClient 上游实现？
    │       ├─ 是 → 使用 tc_ 前缀，放入 config_variables_qmclient.h
    │       └─ 否 ↓
    │
    ├─→ 该功能是否由 QmClient 实现？
    │       ├─ 是 → 使用 qm_ 前缀，放入 config_variables_qimeng.h
    │       └─ 否 ↓
    │
    └─→ 该功能是否属于 DDNet 核心？
            ├─ 是 → 使用对应功能域前缀，放入 config_variables.h（不建议修改上游文件）
            └─ 否 → 使用 qm_ 前缀，放入 config_variables_qimeng.h
```

### 实际案例

| 功能 | 实现者 | 前缀 | 文件 |
|---|---|---|---|
| 腾讯云翻译 | TClient | `tc_translate_*` | `config_variables_qmclient.h` |
| 智谱AI翻译后端 | QmClient | `qm_translate_zhipuai_*` | `config_variables_qimeng.h` |
| 翻译按钮 | QmClient | `qm_translate_show_button` | `config_variables_qimeng.h` |
| 出站翻译 | QmClient | `qm_translate_outgoing_auto` | `config_variables_qimeng.h` |
| 语音聊天 | QmClient | `qm_voice_*` | `config_variables_qimeng.h` |
| 专注模式 | QmClient | `qm_focus_mode_*` | `config_variables_qimeng.h` |
| 饼菜单 | QmClient | `qm_pie_menu_*` | `config_variables_qimeng.h` |
