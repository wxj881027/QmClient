# QmClient 网络卡顿可视化功能报告

## 一、Ping/延迟显示

### 1.1 服务器浏览器中的 Ping 显示

**文件**: `menus_browser.cpp`

- `FormatServerbrowserPing`: 格式化服务器浏览器中的 Ping 值，区分估算延迟和实测延迟
- `GetPingTextColor`: 根据 Ping 值返回颜色，使用 HSL 色彩映射（0-300ms 映射到绿-红）
- 服务器列表有 `COL_PING` 列，支持按 Ping 排序
- 服务器详情面板显示 Ping 值，受 `g_Config.m_UiColorizePing` 控制是否着色
- 好友列表也显示好友所在服务器的延迟

### 1.2 计分板中的 Ping 显示

**文件**: `scoreboard.cpp`

- 计分板表头定义 "Ping" 列
- 每个玩家行显示 `pInfo->m_Latency`，受 `g_Config.m_ClEnablePingColor` 控制是否着色
- 颜色算法：`ColorHSLA((300 - clamp(ping, 0, 300)) / 1000, 1.0, 0.5)`

### 1.3 状态栏中的 Ping 显示

**文件**: `statusbar.cpp` / `statusbar.h`

- `PingRender`: 显示 `pInfo->m_Latency`（当前玩家/被观察玩家的 Ping）
- `PredictionRender`: 显示 `Client()->GetPredictionTime()`（预测时间，毫秒）
- 状态栏默认布局为 `"ac pf r"`，其中 `p` 代表 Ping，`f` 代表 FPS，`d` 代表 Prediction

### 1.4 名字板上的 Ping 圆环

**文件**: `nameplates.cpp`

- `CNamePlatePartPing` 类在名字板旁显示 Ping 彩色圆环
- 受 `g_Config.m_TcNameplatePingCircle` 控制

### 1.5 HUD 中的网络信息

**文件**: `hud.cpp`

- `RenderTextInfo`: 显示 FPS（`cl_showfps`）和预测时间（`cl_showpred`），**不显示 Ping**
- `RenderConnectionWarning`: 当 `Client()->ConnectionProblems()` 为 true 时，显示 "Connection Problems..." 警告文字
- **注释掉的代码**：`// str_format(aBuf, sizeof(aBuf), "%d", g_Config.m_ClWhatsMyPing);`，说明曾经有过在 HUD 中显示 Ping 的尝试，但目前被注释掉了

## 二、FPS 计数器与预测时间

### 2.1 HUD FPS 显示

**文件**: `hud.cpp`

- 使用 `1.0f / Client()->FrameTimeAverage()` 计算 FPS
- 受 `g_Config.m_ClShowfps` 控制
- 支持 V2 动画布局，带淡入淡出效果

### 2.2 状态栏 FPS 显示

**文件**: `statusbar.cpp`

- 使用 `1.0f / m_FrameTimeAverage` 计算 FPS（自身维护的平滑值）
- 字母标识为 `f`

### 2.3 预测时间 (Prediction Time)

**文件**: `client.cpp`

```cpp
int CClient::GetPredictionTime()
{
    int64_t Now = time_get();
    return (int)((m_PredictedTime.Get(Now) - m_aGameTime[g_Config.m_ClDummy].Get(Now)) * 1000 / (float)time_freq());
}
```

- 预测时间 = 客户端预测时间 - 服务器游戏时间的差值，单位毫秒
- 在 HUD（`cl_showpred`）和状态栏（`d` 项）中均可显示
- 引擎注释说明：**Snapshot Latency = 下行延迟，Prediction Latency = 上行延迟**

## 四、Ping 数据来源分析

### 4.1 服务器浏览器 Ping

- `CServerInfo::m_Latency`: 服务器浏览器中的 Ping 值
- `CServerInfo::m_LatencyIsEstimated`: 标识 Ping 是实测还是估算
- 估算通过 `EstimateLatency()` 基于地理位置计算
- 实测通过 ping/pong 机制获取

### 4.2 游戏内玩家 Ping

- 服务端计算每个客户端的延迟：`m_aClients[ClientId].m_Latency = (int)(((time_get() - TagTime) * 1000) / time_freq())`
- 通过 `CNetObj_PlayerInfo::m_Latency` 网络对象发送给客户端
- 这是**往返延迟 (RTT)**，不区分上行/下行

### 4.3 连接问题检测

```cpp
bool CClient::ConnectionProblems() const
{
    return m_aNetClient[g_Config.m_ClDummy].GotProblems(MaxLatencyTicks() * time_freq() / GameTickSpeed());
}
```

- 仅检测"是否长时间未收到数据包"，即**连接是否中断**
- 不提供丢包率、choke 等细粒度指标

### 4.4 NETSTATS 结构

```cpp
typedef struct NETSTATS
{
    uint64_t sent_packets;
    uint64_t sent_bytes;
    uint64_t recv_packets;
    uint64_t recv_bytes;
} NETSTATS;
```

- 仅统计发送/接收的数据包和字节数
- **没有丢包率、choke、重传等指标**

| AntiPing 系列配置 | config_variables.h | 完整 |

### 缺失的功能 ❌（服务器卡顿 vs 客户端卡顿区分）

1. **无上行/下行延迟分离显示**
   - 当前 `m_Latency` 是 RTT（往返延迟），无法区分上行和下行延迟
   - 引擎注释中提到了 "Snapshot Latency = 下行延迟, Prediction Latency = 上行延迟" 的概念，但**未在 UI 中分别展示**

2. **无丢包率 (Packet Loss) 显示**
   - `NETSTATS` 结构只有 `sent_packets/recv_packets`，没有丢包计数
   - 网络层 `CNetConnection` 有重发缓冲区，但未暴露丢包统计
   - UI 中完全没有丢包率指标

3. **无 Choke 指示器**
   - 代码中完全没有 "choke" 概念的实现
   - 不像 Source 引擎那样区分 choke（客户端处理不过来）和 loss（网络丢包）

4. **无服务器 Tick Rate 显示**
   - 客户端知道 `SERVER_TICK_SPEED`（固定值 50），但不显示服务器的实际 tick rate
   - 没有检测服务器是否因性能问题降低 tick rate 的机制

5. **无延迟抖动 (Jitter) 显示**
   - 引擎内部有 `m_AutoMarginLatencyJitterMs` 用于自动边距计算，但**未暴露给 UI**
   - 状态栏和 HUD 都不显示抖动值

6. **无延迟趋势/历史图表**
   - 没有类似 NetGraph 的延迟历史可视化
   - 引擎有 `m_FpsGraph`（FPS 图表），但没有对应的网络延迟图表

7. **连接问题检测过于粗糙**
   - `ConnectionProblems()` 只判断是否长时间未收到数据包（二值：有问题/没问题）
   - 没有中间状态（如"延迟偏高"、"抖动较大"等）

## 六、实现"区分服务器卡顿与客户端卡顿"的建议

### 数据层

1. **在 `CNetConnection` 中添加统计**：丢包计数、重传计数、延迟抖动
2. **分离上行/下行延迟**：利用现有的 ping/pong 机制，在服务端分别记录上行和下行时间
3. **暴露 `m_AutoMarginLatencyAverageMs` 和 `m_AutoMarginLatencyJitterMs`** 到 `IClient` 接口

### UI 层

1. **HUD 增强**：在 `RenderTextInfo()` 中添加 Ping、Jitter、丢包率显示

### 7.3 网络分析器实现

```rust
// src/network_stats/analyzer.rs

use std::collections::VecDeque;

/// 网络质量分析器
pub struct NetworkAnalyzer {
    // 历史数据
    ping_history: VecDeque<u32>,
    upstream_history: VecDeque<u32>,
    downstream_history: VecDeque<u32>,
    loss_history: VecDeque<f32>,
    
    // 配置
    history_size: usize,
    spike_threshold: u32,       // 尖峰阈值 (ms)
    spike_counter: u32,         // 尖峰计数器
    
    // 统计缓存
    cached_stats: Option<NetworkStatistics>,
    last_update: std::time::Instant,
}

impl NetworkAnalyzer {
    pub fn new(history_size: usize) -> Self {
        Self {
            ping_history: VecDeque::with_capacity(history_size),
            upstream_history: VecDeque::with_capacity(history_size),
            downstream_history: VecDeque::with_capacity(history_size),
            loss_history: VecDeque::with_capacity(history_size),
            history_size,
            spike_threshold: 50,
            spike_counter: 0,
            cached_stats: None,
            last_update: std::time::Instant::now(),
        }
    }
    
    /// 添加采样点
    pub fn add_sample(&mut self, sample: NetworkSample) {
        // 检测尖峰
        if let Some(&last_ping) = self.ping_history.back() {
            if sample.ping_ms > last_ping + self.spike_threshold {
                self.spike_counter = self.spike_counter.saturating_add(5);
            } else {
                self.spike_counter = self.spike_counter.saturating_sub(1);
            }
        }
        
        // 添加到历史
        self.ping_history.push_back(sample.ping_ms);
        self.upstream_history.push_back(sample.upstream_ms);
        self.downstream_history.push_back(sample.downstream_ms);
        
        // 计算丢包率
        let loss_rate = if sample.packets_sent > 0 {
            sample.packets_lost as f32 / sample.packets_sent as f32
        } else {
            0.0
        };
        self.loss_history.push_back(loss_rate);
        
        // 保持历史大小
        while self.ping_history.len() > self.history_size {
            self.ping_history.pop_front();
            self.upstream_history.pop_front();
            self.downstream_history.pop_front();
            self.loss_history.pop_front();
        }
        
        // 更新缓存
        self.cached_stats = None;
        self.last_update = std::time::Instant::now();
    }
    
    /// 计算统计数据
    pub fn calculate_statistics(&mut self) -> NetworkStatistics {
        if let Some(ref stats) = self.cached_stats {
            return stats.clone();
        }
        
        let ping_avg = self.calculate_average(&self.ping_history);
        let jitter = self.calculate_jitter(&self.ping_history);
        let upstream_avg = self.calculate_average(&self.upstream_history);
        let downstream_avg = self.calculate_average(&self.downstream_history);
        let packet_loss_rate = self.calculate_average(&self.loss_history);
        
        let ping_min = self.ping_history.iter().copied().min().unwrap_or(0);
        let ping_max = self.ping_history.iter().copied().max().unwrap_or(0);
        
        // 计算质量评分
        let quality_score = self.calculate_quality_score(
            ping_avg, jitter, packet_loss_rate, upstream_avg, downstream_avg
        );
        
        // 确定质量等级
        let quality_level = self.determine_quality_level(quality_score);
        
        let stats = NetworkStatistics {
            ping_avg,
            ping_min,
            ping_max,
            jitter,
            upstream_avg,
            downstream_avg,
            packet_loss_rate,
            resend_rate: 0.0, // TODO: 从采样点计算
            quality_score,
            quality_level,
        };
        
        self.cached_stats = Some(stats.clone());
        stats
    }
    
    /// 计算平均值
    fn calculate_average(&self, data: &VecDeque<u32>) -> f32 {
        if data.is_empty() {
            return 0.0;
        }
        data.iter().sum::<u32>() as f32 / data.len() as f32
    }
    
    /// 计算平均值 (f32)
    fn calculate_average_f32(&self, data: &VecDeque<f32>) -> f32 {
        if data.is_empty() {
            return 0.0;
        }
        data.iter().sum::<f32>() / data.len() as f32
    }
    
    /// 计算抖动 (标准差)
    fn calculate_jitter(&self, data: &VecDeque<u32>) -> f32 {
        if data.len() < 2 {
            return 0.0;
        }
        
        let mean = self.calculate_average(data);
        let variance: f32 = data.iter()
            .map(|&x| {
                let diff = x as f32 - mean;
                diff * diff
            })
            .sum::<f32>() / data.len() as f32;
        
        variance.sqrt()
    }
    
    /// 计算质量评分 (0-100)
    fn calculate_quality_score(
        &self,
        ping: f32,
        jitter: f32,
        loss: f32,
        upstream: f32,
        downstream: f32,
    ) -> u8 {
        // 评分权重
        const PING_WEIGHT: f32 = 0.35;
        const JITTER_WEIGHT: f32 = 0.25;
        const LOSS_WEIGHT: f32 = 0.25;
        const BALANCE_WEIGHT: f32 = 0.15;
        
        // Ping 评分 (0-100ms = 100分, 500ms+ = 0分)
        let ping_score = ((100.0 - ping.min(500.0)) / 100.0 * 100.0).max(0.0);
        
        // Jitter 评分 (0ms = 100分, 100ms+ = 0分)
        let jitter_score = ((50.0 - jitter.min(100.0)) / 50.0 * 100.0).max(0.0);
        
        // 丢包评分 (0% = 100分, 10%+ = 0分)
        let loss_score = ((0.05 - loss.min(0.1)) / 0.05 * 100.0).max(0.0);
        
        // 上行/下行平衡评分 (差异越小越好)
        let balance = (upstream - downstream).abs();
        let balance_score = ((50.0 - balance.min(100.0)) / 50.0 * 100.0).max(0.0);
        
        let total = ping_score * PING_WEIGHT
            + jitter_score * JITTER_WEIGHT
            + loss_score * LOSS_WEIGHT
            + balance_score * BALANCE_WEIGHT;
        
        total.round() as u8
    }
    
    /// 确定质量等级
    fn determine_quality_level(&self, score: u8) -> QualityLevel {
        match score {
            90..=100 => QualityLevel::Excellent,
            70..=89 => QualityLevel::Good,
            50..=69 => QualityLevel::Fair,
            30..=49 => QualityLevel::Poor,
            1..=29 => QualityLevel::Critical,
            _ => QualityLevel::Disconnected,
        }
    }
    
    /// 获取 NetGraph 数据
    pub fn get_graph_data(&self, graph_type: GraphType) -> Vec<GraphDataPoint> {
        let history: &VecDeque<u32> = match graph_type {
            GraphType::Ping => &self.ping_history,
            GraphType::Upstream => &self.upstream_history,
            GraphType::Downstream => &self.downstream_history,
        };
        
        let len = history.len();
        history.iter().enumerate().map(|(i, &value)| {
            let time_offset = ((len - i - 1) * 100) as i32; // 每 100ms 一个点
            let color = self.get_graph_color(value, graph_type);
            GraphDataPoint {
                time_offset_ms: time_offset,
                value: value as f32,
                color,
            }
        }).collect()
    }
    
    /// 获取图表颜色
    fn get_graph_color(&self, value: u32, graph_type: GraphType) -> GraphColor {
        // 根据值和类型返回颜色
        let (r, g, b) = match graph_type {
            GraphType::Ping => {
                // 绿 -> 黄 -> 红
                if value < 100 {
                    (0.0, 1.0, 0.0) // 绿
                } else if value < 200 {
                    (1.0, 1.0, 0.0) // 黄
                } else {
                    (1.0, 0.0, 0.0) // 红
                }
            }
            GraphType::Upstream => (0.0, 0.5, 1.0), // 蓝
            GraphType::Downstream => (0.5, 0.0, 1.0), // 紫
        };
        
        GraphColor { r, g, b, a: 0.75 }
    }
}

pub enum GraphType {
    Ping,
    Upstream,
    Downstream,
}
```

### 7.4 自适应 AntiPing 推荐

```rust
// src/network_stats/antiping.rs

use super::{NetworkStatistics, QualityLevel};

/// AntiPing 推荐配置
/// 注意：limit 和 percent 是互斥的，通过 use_limit_mode 区分
#[repr(C)]
pub struct AntiPingRecommendation {
    pub enabled: bool,           // 是否启用 AntiPing
    pub use_limit_mode: bool,    // true = 使用 limit，false = 使用 percent
    pub limit: u32,              // cl_antiping_limit 推荐值（仅 use_limit_mode=true 时有效）
    pub percent: u32,            // cl_antiping_percent 推荐值（仅 use_limit_mode=false 时有效）
    pub smooth: bool,            // cl_antiping_smooth 推荐值
    pub players: bool,           // cl_antiping_players 推荐值
    pub confidence: f32,         // 推荐置信度 (0.0 - 1.0)
    pub reason: RecommendationReason,
}

#[repr(C)]
#[derive(Clone, Copy)]
pub enum RecommendationReason {
    LowLatency,       // 低延迟，不需要 AntiPing
    ModerateLatency,  // 中等延迟，适度预测
    HighLatency,      // 高延迟，激进预测
    HighJitter,       // 高抖动，保守预测
    PacketLoss,       // 丢包严重，保守预测
    Unstable,         // 网络不稳定
}

impl NetworkAnalyzer {
    /// 推荐 AntiPing 配置
    /// 注意：limit 和 percent 是互斥的，通过 use_limit_mode 区分
    pub fn recommend_antiping(&self) -> AntiPingRecommendation {
        let stats = self.calculate_statistics();
        
        match stats.quality_level {
            QualityLevel::Excellent => AntiPingRecommendation {
                enabled: false,
                use_limit_mode: false,
                limit: 0,
                percent: 100,
                smooth: false,
                players: true,
                confidence: 0.95,
                reason: RecommendationReason::LowLatency,
            },
            
            QualityLevel::Good => {
                // 中等延迟，使用 Percent 模式
                AntiPingRecommendation {
                    enabled: true,
                    use_limit_mode: false,  // 使用 Percent 模式
                    limit: 0,               // 必须为 0
                    percent: 80,
                    smooth: false,
                    players: true,
                    confidence: 0.85,
                    reason: RecommendationReason::ModerateLatency,
                }
            },
            
            QualityLevel::Fair => {
                // 较高延迟或抖动，使用 Limit 模式
                let limit = if stats.jitter > 30.0 {
                    (stats.ping_avg * 0.4) as u32
                } else {
                    (stats.ping_avg * 0.3) as u32
                };
                AntiPingRecommendation {
                    enabled: true,
                    use_limit_mode: true,   // 使用 Limit 模式
                    limit,
                    percent: 0,             // 被忽略
                    smooth: stats.jitter > 20.0,
                    players: true,
                    confidence: 0.70,
                    reason: if stats.jitter > 30.0 {
                        RecommendationReason::HighJitter
                    } else {
                        RecommendationReason::HighLatency
                    },
                }
            },
            
            QualityLevel::Poor => {
                // 网络状况差，使用 Limit 模式
                let limit = (stats.ping_avg * 0.5).min(300.0) as u32;
                AntiPingRecommendation {
                    enabled: true,
                    use_limit_mode: true,   // 使用 Limit 模式
                    limit,
                    percent: 0,             // 被忽略
                    smooth: true,
                    players: stats.jitter < 50.0,
                    confidence: 0.50,
                    reason: if stats.packet_loss_rate > 0.03 {
                        RecommendationReason::PacketLoss
                    } else {
                        RecommendationReason::Unstable
                    },
                }
            },
            
            QualityLevel::Critical | QualityLevel::Disconnected => {
                // 严重问题，使用 Limit 模式
                AntiPingRecommendation {
                    enabled: true,
                    use_limit_mode: true,   // 使用 Limit 模式
                    limit: (stats.ping_avg * 0.7).min(500.0) as u32,
                    percent: 0,             // 被忽略
                    smooth: true,
                    players: false,
                    confidence: 0.30,
                    reason: RecommendationReason::Unstable,
                }
            },
        }
    }
}
```

```cpp
void CHud::RenderNetGraph()
{
    if(!g_Config.m_ClShowNetGraph)
        return;
    
    CNetworkStats* pStats = Client()->GetNetworkStats();
    const auto& graphData = pStats->GetPingGraphData();
    
    // 渲染图表背景
    float graphX = m_Width - 200;
    float graphY = m_Height - 100;
    float graphW = 180;
    float graphH = 80;
    
    Graphics()->TextureClear();
    Graphics()->QuadsBegin();
    Graphics()->SetColor(0.0f, 0.0f, 0.0f, 0.5f);
    IGraphics::CQuadItem Quad(graphX, graphY, graphW, graphH);
    Graphics()->QuadsDrawTL(&Quad, 1);
    Graphics()->QuadsEnd();
    
    // 渲染数据线
    Graphics()->LinesBegin();
    for(size_t i = 1; i < graphData.size(); i++) {
        const auto& prev = graphData[i - 1];
        const auto& curr = graphData[i];
        
        float x1 = graphX + (prev.time_offset_ms / 10000.0f) * graphW;
        float x2 = graphX + (curr.time_offset_ms / 10000.0f) * graphW;
        float y1 = graphY + graphH - (prev.value / 300.0f) * graphH;
        float y2 = graphY + graphH - (curr.value / 300.0f) * graphH;
        
        Graphics()->SetColor(curr.color.r, curr.color.g, curr.color.b, curr.color.a);
        IGraphics::CLineItem Line(x1, y1, x2, y2);
        Graphics()->LinesDraw(&Line, 1);
    }
    Graphics()->LinesEnd();
}
```
