# QmClient 武器切换甩枪动画功能报告

## 一、功能概述

为武器切换添加甩枪过渡动画效果，提升视觉表现。动画需网络同步，确保同客户端的其他玩家能看到。参照本地粒子/远程粒子的实现方式，提供两个独立开关。

```cpp
// 本地粒子：仅渲染本地玩家
if(g_Config.m_QmcFootParticles)
{
    for(int ClientId = 0; ClientId < MAX_CLIENTS; ClientId++)
    {
        if(!IsLocalClient) continue;  // 仅本地

    }

// 远程粒子：仅渲染其他已识别的 QmClient 玩家

    for(int ClientId = 0; ClientId < MAX_CLIENTS; ClientId++)
    {
        if(IsLocalClient) continue;  // 排除本地

    }
}
```

### 3.3 网络同步机制

粒子开关状态通过 **HTTP API** 与 QmClient 服务器同步，**不通过游戏网络快照**：

1. **上报**：客户端将自己的粒子开关状态通过 JSON API 上报
2. **接收**：从服务器获取其他玩家的标记信息
3. **标记**：`MarkQ1menGSyncClient()` 存储到 `m_aQ1menGSyncFootParticlesEnabled[ClientId]`
4. **判断**：`ShouldRenderQ1menGRemoteFootParticles()` 检查双方开关

## 四、甩枪动画方案设计

### 4.1 动画效果描述

"甩枪"动画是指武器切换时，旧武器向一侧甩出消失，新武器从另一侧甩入出现的过渡效果：

```
切换前：  🧑 ── [手枪]
切换中：  🧑 ── [手枪↗] ... [霰弹枪↙]
切换后：  🧑 ── [霰弹枪]
```

动画参数：
- 持续时间：0.15-0.25 秒
- 旧武器：向当前朝向方向旋转飞出 + 缩小 + 淡出
- 新武器：从反方向旋转飞入 + 放大 + 淡入
- 可选：切换瞬间的闪光/粒子效果

### 4.2 数据需求

甩枪动画需要检测**武器切换事件**，即当前帧的 `m_Weapon` 与上一帧不同：

```cpp
// 在 RenderPlayer() 中
int PrevWeapon = m_aPrevWeapon[ClientId];  // 需要新增缓存
int CurWeapon = Player.m_Weapon;

if(PrevWeapon != CurWeapon && PrevWeapon >= 0)
{
    // 触发甩枪动画
    m_aWeaponSwitchTime[ClientId] = Client()->GlobalTime();
    m_aPrevWeaponId[ClientId] = PrevWeapon;
}
m_aPrevWeapon[ClientId] = CurWeapon;
```

### 4.3 动画实现

```cpp
float SwitchProgress = (Client()->GlobalTime() - m_aWeaponSwitchTime[ClientId]) / SWITCH_DURATION;
if(SwitchProgress < 1.0f)
{
    float t = SwitchProgress;

    // 旧武器（前半段动画）
    if(t < 0.5f)
    {
        float OldT = t / 0.5f;  // 0→1
        float OldAngle = Direction.x > 0 ? OldT * pi * 0.5f : -OldT * pi * 0.5f;
        float OldAlpha = 1.0f - OldT;
        float OldScale = 1.0f - OldT * 0.3f;
        // 渲染旧武器精灵，应用 OldAngle, OldAlpha, OldScale
    }

    // 新武器（后半段动画）
    if(t >= 0.3f)
    {
        float NewT = (t - 0.3f) / 0.7f;  // 0→1
        float NewAngle = Direction.x > 0 ? (1.0f - NewT) * -pi * 0.5f : (1.0f - NewT) * pi * 0.5f;
        float NewAlpha = NewT;
        float NewScale = 0.7f + NewT * 0.3f;
        // 渲染新武器精灵，应用 NewAngle, NewAlpha, NewScale
    }
}
```

### 4.4 本地/远程开关

参照粒子系统的模式：

| 变量 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `qm_weapon_switch_anim` | INT | 0 | 本地甩枪动画（0=关, 1=开） |
| `qm_weapon_switch_anim_remote` | INT | 0 | 远程甩枪动画（0=关, 1=开） |

渲染逻辑：

```cpp
// 本地甩枪动画
if(g_Config.m_QmWeaponSwitchAnim)
{
    if(IsLocalClient)
        RenderWeaponSwitchAnim(ClientId, ...);
}

// 远程甩枪动画
if(g_Config.m_QmWeaponSwitchAnimRemote)
{
    if(!IsLocalClient && GameClient()->IsQ1menGClientRecognized(ClientId))
        RenderWeaponSwitchAnim(ClientId, ...);
}
```

### 4.5 网络同步

**方案 A：复用粒子系统的 HTTP API 同步**

在现有的 QmClient 中心服同步中添加甩枪动画开关字段：

```json
// 上报
{
    "foot_particles_enabled": true,
    "remote_particles_enabled": true,
    "weapon_switch_anim_enabled": true   // 新增
}

// 接收
Mark.m_WeaponSwitchAnimEnabled = JsonReadBoolean(...)
```

**优势**：无需修改游戏网络协议，与粒子系统共用同步通道。

**方案 B：利用现有网络快照**

武器切换事件已经通过 `CNetObj_Character.m_Weapon` 字段同步到所有客户端。远程客户端可以检测 `m_Weapon` 的变化来触发动画，**不需要额外的网络消息**。

但需要知道对方是否开启了甩枪动画，才能决定是否渲染。这仍然需要方案 A 的同步机制。

**推荐**：方案 A + B 结合。武器切换事件从现有快照检测，开关状态从 HTTP API 同步。

### 4.6 动画风格选项

| 变量 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `qm_weapon_switch_anim_style` | INT | 0 | 动画风格（0=甩枪, 1=缩放, 2=旋转） |

| 风格 | 旧武器效果 | 新武器效果 |
|------|-----------|-----------|
| 甩枪 | 向朝向方向旋转飞出 | 从反方向旋转飞入 |
| 缩放 | 缩小+淡出 | 从小放大+淡入 |
| 旋转 | 绕中心旋转半圈消失 | 继续旋转半圈出现 |

## 六、技术风险与注意事项

### 6.1 武器切换检测的准确性

远程玩家的武器切换需要从快照中检测 `m_Weapon` 变化。由于快照有插值（`Intra`），可能出现：
- 同一帧内多次切换只检测到最终状态
- 快照丢失导致切换事件遗漏

**缓解**：缓存上一帧的 `m_Weapon` 值，每帧比较。

### 6.2 与 Jelly Tee 的兼容

Jelly Tee 系统已经修改了 `BodyAngle` 和 `FeetAngle`，甩枪动画的旋转效果需要与之协调，避免叠加导致视觉异常。

### 6.3 性能考虑

甩枪动画需要额外的武器精灵渲染（旧武器+新武器同时渲染），但持续时间很短（0.15-0.25秒），对性能影响可忽略。
