# QmClient 配置导入/导出系统功能报告

## 一、功能概述

配置导入/导出系统旨在将 QmClient 分散在多个文件中的配置统一打包，支持一键导出为归档文件、选择性导出、智能合并导入，方便用户备份、迁移和分享配置。

---

## 二、当前配置系统架构

### 2.1 配置域（ConfigDomain）系统

QmClient 使用 X-Macro 模式定义 5 个配置域：

| 枚举值 | 文件名 | HasVars | 说明 |
|--------|--------|---------|------|
| `DDNET` | `settings_ddnet.cfg` | true | DDNet 原生配置变量 |
| `QIMENG` | `settings_qmclient.cfg` | true | QmClient/TClient 配置变量 |
| `TCLIENTPROFILES` | `qmclient_profiles.cfg` | false | 皮肤配置方案（仅回调写入） |
| `TCLIENTCHATBINDS` | `qmclient_chatbinds.cfg` | false | 聊天绑定（仅回调写入） |
| `TCLIENTWARLIST` | `qmclient_warlist.cfg` | false | 敌对列表（仅回调写入） |

定义在 [config_domains.h](file:///e:/Coding/DDNet/QmClient/src/engine/shared/config_domains.h)，通过两次 include 分别生成枚举和元数据数组。

### 2.2 配置变量包含链

[config_includes.h](file:///e:/Coding/DDNet/QmClient/src/engine/shared/config_includes.h) 控制变量归属：

```cpp
SET_CONFIG_DOMAIN(ConfigDomain::DDNET)
#include "config_variables.h"          // ~200+ DDNet 变量

SET_CONFIG_DOMAIN(ConfigDomain::QIMENG)
#include "config_variables_qmclient.h" // ~236 TClient 变量

SET_CONFIG_DOMAIN(ConfigDomain::QIMENG)
#include "config_variables_qimeng.h"   // ~194 栖梦变量
```

**注意**：`config_variables_qmclient.h` 和 `config_variables_qimeng.h` 都归属 `ConfigDomain::QIMENG`，保存到同一个文件 `settings_qmclient.cfg`。

### 2.3 三种配置变量类型

| 宏类型 | C++ 成员 | CFG 格式 | 示例 |
|--------|----------|----------|------|
| `MACRO_CONFIG_INT` | `int m_##Name` | `script_name 42` | `cl_predict 1` |
| `MACRO_CONFIG_COL` | `unsigned m_##Name` | `script_name 4294967295` | `cl_color 0xFF00FF` |
| `MACRO_CONFIG_STR` | `char m_##Name[Len]` | `script_name "value"` | `player_name "tee"` |

### 2.4 配置文件保存机制

保存逻辑在 [CConfigManager::Save()](file:///e:/Coding/DDNet/QmClient/src/engine/shared/config.cpp#L379-L478)：

1. 为每个域打开 `.tmp` 临时文件（原子写入）
2. 写入非默认值且带 `CFGFLAG_SAVE` 标志的变量
3. 执行回调函数写入自定义数据
4. 仅 DDNET 域写入未知命令
5. sync + close + rename 临时文件为正式文件

### 2.5 配置文件加载机制

客户端启动时按序加载（[client.cpp:5250-5279](file:///e:/Coding/DDNet/QmClient/src/engine/client/client.cpp#L5250-L5279)）：

1. `settings_ddnet.cfg`（DDNET 域）
2. `settings_qmclient.cfg`（QIMENG 域）
3. `qmclient_profiles.cfg`（TCLIENTPROFILES 域）
4. `qmclient_chatbinds.cfg`（TCLIENTCHATBINDS 域）
5. `qmclient_warlist.cfg`（TCLIENTWARLIST 域）
6. `autoexec_client.cfg` 或 `autoexec.cfg`（回退）
7. 命令行参数

每行作为控制台命令执行（`ExecuteLine`），有递归保护。

### 2.6 各域回调写入内容

| 域 | 注册组件 | 写入内容 |
|----|---------|---------|
| `DDNET` | CSkins | 收藏皮肤 (`add_favorite_skin`) |
| `DDNET` | IFavorites | 收藏服务器 (`add_favorite`) |
| `DDNET` | CQmClient | 收藏地图 (`add_favorite_map`) |
| `QIMENG` | CSkins | 皮肤队列/预设 (`add_skin_queue_*`) |
| `QIMENG` | CBindWheel | 绑定轮 (`add_bindwheel`) |
| `TCLIENTPROFILES` | CSkinProfiles | 皮肤方案 (`add_profile`) |
| `TCLIENTCHATBINDS` | CBindChat | 聊天绑定 (`bindchat`/`unbindchat`) |
| `TCLIENTWARLIST` | CWarList | 敌对列表 (`update_war_group`/`add_war_entry`) |

### 2.7 用户自定义配置文件

- `autoexec_client.cfg` / `autoexec.cfg`：用户自定义启动脚本
- 通过 `exec <filename>` 手动加载任意 cfg 文件
- 通过 `+toggle_exec <press_file> <release_file>` 按键执行文件
- 用户可能创建自定义 cfg 文件夹存放按键绑定和高级操作

### 2.8 其他配置相关文件

| 文件 | 位置 | 说明 |
|------|------|------|
| `qmclient/map_categories.json` | save 目录 | 地图分类缓存（JSON） |
| `qmclient/workshop/` | save 目录 | Workshop 缩略图和列表缓存 |
| `ddnet-info.json` | save 目录 | DDNet 服务器信息缓存 |
| `storage.cfg` | 数据目录 | 存储路径配置 |

## 四、导入/导出方案设计

### 4.1 归档格式：`.qmconfig`

采用 **ZIP 格式**（与 `.apk`、`.docx` 等相同），扩展名为 `.qmconfig`：

```
config-export-2026-04-19.qmconfig
├── manifest.json          # 元数据
├── settings_ddnet.cfg     # DDNet 配置
├── settings_qmclient.cfg  # QmClient 配置
├── qmclient_profiles.cfg  # 皮肤方案
├── qmclient_chatbinds.cfg # 聊天绑定
├── qmclient_warlist.cfg   # 敌对列表
├── autoexec_client.cfg    # 用户启动脚本（可选）
└── user_configs/          # 用户自定义 cfg 文件夹（可选）
    └── *.cfg
```

### 4.2 manifest.json 元数据

```json
{
    "version": 1,
    "export_time": "2026-04-19T12:00:00Z",
    "client_version": "2.35.0",
    "git_revision": "abc12345",
    "export_options": {
        "include_ddnet": true,
        "include_qmclient": true,
        "include_profiles": true,
        "include_chatbinds": true,
        "include_warlist": true,
        "include_autoexec": true,
        "include_user_configs": false
    },
    "file_hashes": {
        "settings_ddnet.cfg": "sha256:abcdef...",
        "settings_qmclient.cfg": "sha256:123456..."
    }
}
```

### 4.3 选择性导出

导出时用户可勾选要包含的配置类别：

| 类别 | 包含文件 | 默认选中 |
|------|----------|----------|
| DDNet 基础设置 | `settings_ddnet.cfg` 中的变量 | ✅ |
| QmClient 设置 | `settings_qmclient.cfg` 中的变量 | ✅ |
| 皮肤方案 | `qmclient_profiles.cfg` | ✅ |
| 聊天绑定 | `qmclient_chatbinds.cfg` | ✅ |
| 敌对列表 | `qmclient_warlist.cfg` | ✅ |
| 收藏数据 | 收藏皮肤/服务器/地图 | ✅ |
| 启动脚本 | `autoexec_client.cfg` | ❌ |
| 用户自定义 cfg | `user_configs/` 文件夹 | ❌ |

### 4.4 导入时智能合并

导入时提供三种模式：

| 模式 | 行为 | 适用场景 |
|------|------|----------|
| **覆盖** | 导入的配置完全替换当前配置 | 全新安装、迁移到新电脑 |
| **保留** | 保留当前配置，仅添加导入中有但当前没有的条目 | 合并两台电脑的配置 |
| **逐项选择** | 逐个配置项选择使用导入值还是保留当前值 | 精细控制 |

**冲突检测**：对于同一配置变量，如果导入值与当前值不同，标记为冲突项。

**合并规则**：

| 配置类型 | 覆盖模式 | 保留模式 |
|----------|----------|----------|
| INT/COL 变量 | 使用导入值 | 保留当前值 |
| STR 变量 | 使用导入值 | 保留当前值 |
| 收藏列表 | 合并（去重） | 合并（去重） |
| 绑定轮 | 合并（按名称去重） | 保留当前 |
| 敌对列表 | 合并（按名称去重） | 合并（去重） |

### 4.5 版本兼容

- 导入时检查 `manifest.json` 中的 `client_version`
- 如果导入版本比当前版本旧，尝试自动迁移（如变量重命名）
- 如果导入版本比当前版本新，提示"此配置来自更新版本，部分设置可能不兼容"

## 五、实现方案

### 5.1 ZIP 库选择

项目已有 minizip（通过 ddnet-libs 子模块），CMake 中已链接。可直接使用。

### 5.2 导出流程

```
1. 用户点击"导出配置"按钮
2. 弹出选择性导出对话框（勾选类别）
3. 创建临时目录
4. 遍历选中的配置域，将当前配置写入临时文件
   - 对于变量：使用 CConfigManager 的序列化逻辑
   - 对于回调数据：触发回调写入
5. 生成 manifest.json
6. 打包为 ZIP 归档
7. 弹出文件保存对话框
8. 清理临时文件
```

### 5.3 导入流程

```
1. 用户点击"导入配置"按钮
2. 弹出文件选择对话框（.qmconfig 过滤）
3. 解压到临时目录
4. 读取 manifest.json，验证版本
5. 弹出合并模式选择对话框（覆盖/保留/逐项选择）
6. 按选定模式合并配置
7. 提示"需要重启客户端以应用所有更改"
8. 清理临时文件
```
