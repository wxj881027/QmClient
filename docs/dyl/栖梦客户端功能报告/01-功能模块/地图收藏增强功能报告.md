# QmClient 地图收藏增强功能报告

## 一、功能概述

地图收藏增强旨在将当前位于栖梦设置页面中的地图收藏功能提升为浏览器页面的核心功能，与服务器收藏并列，并增强服务器列表中收藏地图的识别和筛选能力。

---

## 二、当前实现状态

### 2.1 两套独立的收藏系统

QmClient 中存在两套收藏系统，架构完全不同：

| 对比项 | 服务器收藏（DDNet 原生） | 地图收藏（QmClient 新增） |
|--------|------------------------|------------------------|
| 接口 | `IFavorites` | `CTClient` 内部方法 |
| 存储键 | `NETADDR` 网络地址 | 地图名字符串 |
| 数据结构 | `CEntry`（地址数组） | `std::set<std::string>` |
| CFG 格式 | `add_favorite <ip:port>` | `add_favorite_map "<map_name>"` |
| CFG 归属 | `settings_ddnet.cfg`（DDNET 域） | `settings_ddnet.cfg`（DDNET 域） |
| UI 位置 | 浏览器页面（星形图标列） | 栖梦设置页面（模块卡片） |

### 2.4 配置持久化

地图收藏保存到 `settings_ddnet.cfg`（DDNET 域），通过 `ConfigSaveFavoriteMaps` 回调写入：

```
add_favorite_map "MapName1"
add_favorite_map "DDmaX Easy SomeMap"
```

### 2.5 地图分类缓存系统

配套的分类信息存储在 `qmclient/map_categories.json`（JSON 格式）：

```json
{
    "version": 1,
    "maps": {
        "MapName1": "DDmaX Easy",
        "MapName2": "Novice"
    }
}
```

延迟保存机制：修改后 5 秒延迟保存，在线游戏且菜单未打开时不执行保存。

## 三、当前问题分析

### 3.1 地图收藏管理入口不显眼

当前地图收藏的管理界面位于栖梦设置页面（SETTINGS_QIMENG）的模块卡片中，用户需要：
1. 打开设置
2. 切换到栖梦标签
3. 找到 "Favorite maps" 模块

这个入口对用户不够直观，尤其是新用户很难发现。

### 3.2 浏览器页面缺少地图收藏筛选

服务器浏览器有 `br_filter_favorite` 筛选收藏服务器，但**没有对应的地图收藏筛选**。用户无法快速找到运行收藏地图的服务器。

### 3.3 地图收藏与服务器收藏割裂

两套收藏系统完全独立，UI 上没有关联。用户在浏览器页面看到星形图标（服务器收藏），但看不到地图收藏的标识（金色地图名需要仔细观察）。

### 3.4 地图收藏缺少评分/备注

当前地图收藏只有"收藏/未收藏"二值状态，没有个人评分、备注等元数据。

## 四、增强方案

### 4.1 浏览器页面增强（高优先级）

#### 4.1.1 地图收藏列

在服务器浏览器列表中新增 `COL_FLAG_FAV_MAP` 列，与服务器收藏星形图标并列：

```
┌──────────────────────────────────────────────────────────────────┐
│ ☆ ★ │ 🗺 │ Server Name        │ Map          │ Players │ Ping  │
│  ★   │ ★ │ DDNet GER          │ Multeasymap  │ 12/64   │ 45    │
│      │   │ DDNet USA          │ Shortmap     │ 8/64    │ 120   │
│  ★   │ ★ │ DDNet JPN          │ Multeasymap  │ 32/64   │ 80    │
└──────────────────────────────────────────────────────────────────┘
  ↑       ↑
  服务器   地图收藏
  收藏     标识
```

- 图标：使用 `FONT_ICON_MAP` 或自定义地图图标
- 颜色：金色 `(1.0, 0.85, 0.0, 1.0)` 与当前金色高亮一致
- 交互：点击图标切换收藏状态

#### 4.1.2 地图收藏筛选

新增浏览器筛选选项：

| 配置变量 | 类型 | 默认值 | 说明 |
|----------|------|--------|------|
| `br_filter_favorite_map` | INT | 0 | 筛选运行收藏地图的服务器 |

在浏览器筛选栏中添加 "Favorite maps" 复选框，与 "Favorites" 并列。

#### 4.1.3 收藏地图独立 Tab

在浏览器页面添加 "Favorite Maps" Tab，与 "Internet" / "LAN" / "Favorites" 并列：

```
[Internet] [LAN] [Favorites] [Favorite Maps]
```

"Favorite Maps" Tab 显示所有运行收藏地图的服务器列表，按地图名分组。

### 4.2 地图收藏管理增强（中优先级）

#### 4.2.1 地图收藏数据结构扩展

```cpp
struct SFavoriteMapEntry
{
    char m_aName[MAX_MAP_LENGTH];
    char m_aNote[128];           // 个人备注
    int m_Rating;                // 1-5 星评分（0=未评分）
    int64_t m_AddedTimestamp;    // 收藏时间
};
```

#### 4.2.2 控制台命令扩展

| 命令 | 参数 | 说明 |
|------|------|------|
| `add_favorite_map` | `s[map_name] [rating]` | 添加收藏（可选评分） |
| `set_favorite_map_note` | `s[map_name] s[note]` | 设置地图备注 |
| `set_favorite_map_rating` | `s[map_name] i[rating]` | 设置地图评分 |

#### 4.2.3 CFG 格式扩展

```
add_favorite_map "MapName1" 5
set_favorite_map_note "MapName1" "很好玩的图"
```

### 4.3 游戏内增强（低优先级）

- 投票列表中收藏地图旁显示评分星星
- 地图完成时自动提示"是否收藏此地图"
- 收藏地图完成率统计
