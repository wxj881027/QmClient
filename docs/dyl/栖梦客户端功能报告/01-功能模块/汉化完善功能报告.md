# QmClient 汉化完善功能报告

## 一、本地化基础设施概览

### 1.1 Localize() 工作原理

`Localize(pStr, pContext="")` 定义在 `src/game/localization.h`，工作流程：

1. 对原始字符串 `pStr` 和上下文 `pContext` 分别计算 DJB2 哈希值
2. 在全局翻译数据库 `g_Localization` 中查找匹配的翻译
3. 如果找到翻译则返回翻译文本，否则返回原始英文字符串（**回退机制**）

**实际代码实现**（[localization.cpp:13-17](file:///e:/Coding/DDNet/QmClient/src/game/localization.cpp#L13-L17)）：

```cpp
const char *Localize(const char *pStr, const char *pContext)
{
    const char *pNewStr = g_Localization.FindString(str_quickhash(pStr), str_quickhash(pContext));
    return pNewStr ? pNewStr : pStr;
}
```

> **设计意图**：`Localize()` 的参数**始终为英文原文**，翻译系统以英文为 key 查找各语言翻译。如果传入中文，则非中文用户永远看到中文。

### 1.3 QmClient 双层语言文件加载机制

**实际代码**（[gameclient.cpp:394-406](file:///e:/Coding/DDNet/QmClient/src/game/client/gameclient.cpp#L394-L406)）：

```cpp
void CGameClient::InitializeLanguage()
{
    g_Localization.LoadIndexfile(Storage(), Console());
    if(g_Config.m_ClShowWelcome)
        g_Localization.SelectDefaultLanguage(Console(), g_Config.m_ClLanguagefile, sizeof(g_Config.m_ClLanguagefile));

    // 第一步：加载 DDNet 基础语言文件 (Clear=true 默认值，清除旧数据)
    g_Localization.Load(g_Config.m_ClLanguagefile, Storage(), Console());

    // 第二步：加载 QmClient 扩展语言文件 (Clear=false，追加到已有数据)
    char aBuf[512];
    str_format(aBuf, sizeof(aBuf), "qmclient/%s", g_Config.m_ClLanguagefile);
    g_Localization.Load(aBuf, Storage(), Console(), false);
}
```

| 加载阶段 | 文件路径 | Clear 参数 | 效果 |
|---------|---------|-----------|------|
| DDNet 基础 | `languages/simplified_chinese.txt` | `true`（默认值） | 清空旧数据，重新加载 |
| QmClient 扩展 | `qmclient/languages/simplified_chinese.txt` | `false` | 追加到已有数据，覆盖同名条目 |

> ⚠️ **QmClient 扩展语言文件尚未创建**。`data/qmclient/` 目录下只有 `builtinscripts/`、`fonts/`、`gui_logo.png`，**没有 `data/qmclient/languages/` 目录**。当前 `Load("qmclient/...", ..., false)` 调用因找不到文件而**静默返回 false**，不会产生错误，但也不会加载任何 QmClient 特有的翻译。

### 1.4 标记函数

**实际代码**（[localization.h](file:///e:/Coding/DDNet/QmClient/src/engine/shared/localization.h)）：

```cpp
static constexpr const char *Localizable(const char *pStr, const char *pContext = "")
{
    return pStr;
}

static constexpr const char *TCLocalizable(const char *pStr, const char *pContext = "")
{
    return pStr;
}
```

| 函数 | 用途 | 运行时行为 |
|------|------|-----------|
| `Localize(pStr, pContext)` | 执行翻译查找 | 返回翻译文本或原文 |
| `Localizable(pStr, pContext)` | 标记可翻译字符串 | 直接返回原文，**编译期零开销** |
| `TCLocalizable(pStr, pContext)` | QmClient 专用标记 | 直接返回原文，**编译期零开销** |

> `Localizable` 用于 DDNet 原有代码，`TCLocalizable` 用于 TaterClient/QmClient 的代码（前缀 TC = TaterClient）。两者功能完全相同，区分是为了在提取翻译时能区分来源。

## 三、服务端本地化 — 几乎无需修改

### 3.1 现状

服务端**完全没有本地化基础设施**：

- 不包含 `<game/localization.h>` 头文件
- 不使用 `Localize()` 函数（零调用）
- 没有独立的语言文件
- 没有 `CLocalizationDatabase` 实例

所有面向玩家的消息都是硬编码英文字符串，QmClient 仅在 1 处替换为硬编码中文。

### 3.2 QmClient 特有的服务端中文硬编码

| 文件 | 行号 | 硬编码中文 |
|------|------|-----------|
| DDRace.cpp | 162 | `"服务器踢人/观战投票已不再由管理员主动监管。"` |

> **注意**：此前报告中统计的服务端硬编码中文数量（~382 处）有误。经实际验证，服务端中 QmClient 特有的中文硬编码仅此 1 处，其余均为英文字符串（来自上游 DDNet）。

### 3.3 服务端本地化的技术障碍

1. 服务端与客户端是分开编译的二进制文件，服务端不链接本地化代码
2. 服务端消息面向所有玩家，不同玩家可能使用不同语言
3. 实现服务端本地化需要：引入 `CLocalizationDatabase`、加载语言文件、根据每个玩家的语言偏好发送翻译后的消息

## 九、翻译文件维护性问题

### 9.1 `simplified_chinese.txt` 重复条目问题

当前简体中文翻译文件存在大量重复条目，即同一个英文原文出现多条翻译，但只有**最后一条**生效（后加载的覆盖先加载的）。

**重复类型**：

| 类型 | 说明 | 示例 |
|------|------|------|
| 完全重复 | 同一英文原文出现多条**相同**的中文翻译 | 浪费维护精力 |
| 翻译冲突 | 同一英文原文出现多条**不同**的中文翻译 | 只有最后一条生效，前面的翻译被静默覆盖 |
| 上下文重复 | 同一英文原文在不同 `[context]` 下有不同翻译 | **这是合理的**，不应删除 |

**翻译冲突示例**（含行号）：

### 9.3 `[context]` 上下文机制详解

#### 9.3.1 机制原理

`[context]` 是翻译文件中的特殊标记，用于解决**同一英文原文在不同场景下需要不同翻译**的问题。

**翻译文件语法**：

```
英文原文
== 中文翻译                    ← 默认翻译（无上下文）

[context名称]
英文原文
== 中文翻译                    ← 在指定上下文中的翻译
```

> **注意**：翻译行以 `== ` 开头，这是语言文件的实际格式。每个条目由"原文行"和"翻译行"两行组成，`[context]` 行出现在原文行之前。

**代码调用方式**：

```cpp
Localize("Save")                    // 使用默认翻译
Localize("Save", "Editor")          // 使用 [Editor] 上下文中的翻译
```

#### 9.3.2 查找优先级

`CLocalizationDatabase::FindString()` 的实际实现（[localization.cpp:261-282](file:///e:/Coding/DDNet/QmClient/src/game/localization.cpp#L261-L282)）：

```cpp
const char *CLocalizationDatabase::FindString(unsigned Hash, unsigned ContextHash) const
{
    CString String;
    String.m_Hash = Hash;
    String.m_ContextHash = ContextHash;
    String.m_pReplacement = nullptr;

    // 第一步：用 (Hash, ContextHash) 精确查找
    auto Range1 = std::equal_range(m_vStrings.begin(), m_vStrings.end(), String);
    if(std::distance(Range1.first, Range1.second) == 1)
        return Range1.first->m_pReplacement;

    // 第二步：如果精确查找未命中，且上下文不是默认的空上下文，则回退到默认上下文查找
    const unsigned DefaultHash = str_quickhash("");
    if(ContextHash != DefaultHash)
    {
        String.m_ContextHash = DefaultHash;
        auto Range2 = std::equal_range(m_vStrings.begin(), m_vStrings.end(), String);
        if(std::distance(Range2.first, Range2.second) == 1)
            return Range2.first->m_pReplacement;
    }

    return nullptr;
}
```

**查找优先级**：

1. **优先精确匹配**：先尝试 `(原文哈希, 上下文哈希)` 的精确二元组查找
2. **回退默认上下文**：如果精确匹配未命中，且传入的上下文不是空字符串，则尝试用 `(原文哈希, 默认上下文哈希)` 查找
3. **最终回退**：如果以上两步都未命中，返回 `nullptr`，由 `Localize()` 回退返回原文

> **重要细节**：`equal_range` 返回的范围中，如果 `distance == 1` 才算命中。如果 `distance > 1`（即存在多个匹配项），也会返回 `nullptr`，这是一种**安全机制**，避免在有歧义时返回错误翻译。这意味着**重复条目不仅浪费空间，还可能导致翻译查找失败**。

**查找流程图**：

```
Localize("Save", "Editor")
    │
    ├─→ 精确查找 (Hash("Save"), Hash("Editor"))
    │   ├─ distance == 1 → 返回 "保存"
    │   ├─ distance > 1  → 歧义！返回 nullptr → 继续回退
    │   └─ distance == 0 → 未找到 → 继续回退
    │
    ├─→ 默认查找 (Hash("Save"), Hash(""))
    │   ├─ distance == 1 → 返回 "储存"
    │   └─ distance != 1 → 继续回退
    │
    └─→ 返回原文 "Save"
```

#### 9.3.3 语言文件加载逻辑

`CLocalizationDatabase::Load()` 的实际实现（[localization.cpp:169-246](file:///e:/Coding/DDNet/QmClient/src/game/localization.cpp#L169-L246)）：

```cpp
bool CLocalizationDatabase::Load(const char *pFilename, IStorage *pStorage, IConsole *pConsole, bool Clear)
{
    // Clear=true 时清除已有数据，Clear=false 时追加
    if(Clear)
    {
        m_vStrings.clear();
        m_StringsHeap.Reset();
    }

    char aContext[512];
    while(const char *pLine = LineReader.Get())
    {
        if(!str_length(pLine)) continue;
        if(pLine[0] == '#') continue;  // 跳过注释

        // [context] 解析逻辑
        if(pLine[0] == '[')
        {
            // 提取 [ 和 ] 之间的内容作为上下文
            str_truncate(aContext, sizeof(aContext), pLine + 1, Len - 2);
            pLine = LineReader.Get();  // 读取下一行（原文行）
        }
        else
        {
            aContext[0] = '\0';  // 非 [ 开头的行重置上下文为空
        }

        // 读取原文和翻译
        str_copy(aOrigin, pLine);
        const char *pReplacement = LineReader.Get();
        // 翻译行必须以 "== " 开头
        // 空翻译 "== " 跳过（表示不翻译此条目）

        AddString(aOrigin, pReplacement + 3, aContext);
    }
    std::sort(m_vStrings.begin(), m_vStrings.end());  // 排序以支持 equal_range 二分查找
}
```

**[context] 解析规则**：
- 语言文件中 `[context_name]` 行表示后续条目属于该上下文
- **只有紧跟在 `[context]` 行后面的那一对原文+翻译才属于该上下文**
- 每次循环开头如果不是 `[` 开头就会重置 `aContext` 为空

#### 9.3.4 重复条目处理机制

`AddString()` 的实际实现（[localization.cpp:248-259](file:///e:/Coding/DDNet/QmClient/src/game/localization.cpp#L248-L259)）：

```cpp
void CLocalizationDatabase::AddString(const char *pOrgStr, const char *pNewStr, const char *pContext)
{
    const unsigned Hash = str_quickhash(pOrgStr);
    const unsigned ContextHash = str_quickhash(pContext);
    const auto It = std::find_if(m_vStrings.begin(), m_vStrings.end(), [Hash, ContextHash](const CString &String) {
        return String.m_Hash == Hash && String.m_ContextHash == ContextHash;
    });
    if(It == m_vStrings.end())
        m_vStrings.emplace_back(Hash, ContextHash, m_StringsHeap.StoreString(*pNewStr ? pNewStr : pOrgStr));
    else
        It->m_pReplacement = m_StringsHeap.StoreString(*pNewStr ? pNewStr : pOrgStr);  // 覆盖已有翻译
}
```

**重复条目处理策略：后加载覆盖先加载**

- `AddString()` 在添加前会先搜索是否已存在相同 `(Hash, ContextHash)` 的条目
- 如果**已存在**，则**直接替换** `m_pReplacement` 指针，指向新翻译
- 如果**不存在**，则添加新条目
- 这意味着：**后加载的翻译会覆盖先加载的同键翻译**

> ⚠️ **重复条目的潜在风险**：虽然 `AddString()` 会覆盖已有条目（不会产生重复），但如果翻译文件中存在重复条目，在 `Load()` 结束后调用 `std::sort()` 排序时，不会产生重复元素。然而，如果**同一个英文原文在同一上下文中出现多次但翻译不同**（翻译冲突），后出现的翻译会覆盖先出现的，这可能导致意外的翻译结果。
