# QmClient 翻译功能分析报告

## 一、功能概述

QmClient 的翻译功能是一个**聊天消息实时翻译系统**，支持将游戏内聊天消息从一种语言翻译为另一种语言。该系统是 QmClient 的特色功能之一，提供手动翻译和自动翻译两种模式。

> ⚠️ 注意：代码库中存在两种"翻译"概念。本报告聚焦于**语言翻译**（聊天消息多语言互译），而非 DDNet 上游的**协议翻译**（Teeworlds 0.7→0.6 协议转换，即 `sixup_translate_*` 系列文件）。

***

## 二、架构设计

### 类层次结构

```
ITranslateBackend (抽象接口)
│   ├── EncodeTarget()     -- 编码目标语言代码
│   ├── CompareTargets()   -- 比较两个目标语言是否相同
│   ├── Name()             -- 返回后端名称
│   └── Update()           -- 轮询翻译进度，返回 std::optional<bool>
│
└── ITranslateBackendHttp (HTTP 请求基类，内部类)
     ├── m_pHttpRequest       -- HTTP 请求智能指针
     ├── CreateHttpRequest()  -- 创建并运行 HTTP 请求
     ├── ParseResponse()      -- 纯虚函数，子类实现 JSON 解析
     └── Update()             -- 统一的 HTTP 状态检查逻辑
          │
          ├── CTranslateBackendTencentCloud   -- 腾讯云翻译后端
          ├── CTranslateBackendLibretranslate  -- LibreTranslate 后端
          ├── CTranslateBackendFtapi           -- FreeTranslateAPI 后端
          └── CTranslateBackendZhipuAI         -- 智谱AI翻译后端 ✅ 已完成
```

### 核心组件

| 类                    | 文件              | 职责                |
| -------------------- | --------------- | ----------------- |
| `CTranslate`         | translate.h/cpp | 翻译组件主类，管理翻译任务生命周期 |
| `ITranslateBackend`  | translate.h/cpp | 翻译后端抽象接口          |
| `CTranslateResponse` | chat.h          | 翻译结果数据结构          |

## 三、翻译后端

### 1. 腾讯云翻译（TencentCloud）— 当前默认后端

| 属性     | 值                                                                      |
| ------ | ---------------------------------------------------------------------- |
| 配置值    | `tencentcloud`                                                         |
| API 服务 | 腾讯云 TMT (Text Translate)                                               |
| 默认端点   | `https://tmt.tencentcloudapi.com/`                                     |
| 认证方式   | TC3-HMAC-SHA256 签名算法（完整 HMAC-SHA256 签名链）                               |
| 请求方式   | POST JSON                                                              |
| 语言编码   | `zh-cn` → `zh`，`zh-tw` → `zh-TW`                                       |
| 凭证获取   | 配置变量 → 环境变量 `TENCENTCLOUD_SECRET_ID/KEY` → 环境变量 `TC_SECRET_ID/KEY` → 空 |
| 费用     | 付费服务，需腾讯云账号充值                                                          |

### 2. LibreTranslate

| 属性     | 值                                    |
| ------ | ------------------------------------ |
| 配置值    | `libretranslate`                     |
| API 服务 | LibreTranslate 开源翻译引擎                |
| 默认端点   | `localhost:5000/translate`（自部署）      |
| 认证方式   | 可选 API Key（通过 `tc_translate_key` 配置） |
| 请求方式   | POST JSON                            |
| 特色     | 支持语言检测置信度验证                          |
| 费用     | 免费自部署；公共实例有速率限制                      |

### 3. FreeTranslateAPI（FTAPI）

| 属性     | 值                         |
| ------ | ------------------------- |
| 配置值    | `ftapi`                   |
| API 服务 | ftapi.pythonanywhere.com  |
| 请求方式   | GET 请求（URL 参数）            |
| 语言编码   | `zh` → `zh-cn`            |
| 限制     | **自动翻译被禁用**（防止过载导致公共服务关闭） |
| 费用     | 免费（公共服务，有速率限制）            |

### 4. 智谱AI翻译（ZhipuAI）— ✅ 已完成

| 属性     | 值                                                       |
| ------ | ------------------------------------------------------- |
| 配置值    | `zhipuai`                                               |
| API 服务 | 智谱AI GLM 大模型 Chat Completions API                       |
| 默认端点   | `https://open.bigmodel.cn/api/paas/v4/chat/completions` |
| 认证方式   | Bearer Token（API Key 直接鉴权）复用 `tc_translate_key`        |
| 请求方式   | POST JSON                                               |
| 默认模型   | `glm-4.7-flash`（免费额度充足）                                 |
| 费用     | **免费额度**，个人用户注册即送免费调用额度                                 |

#### API 调用规范

**请求端点**：`https://open.bigmodel.cn/api/paas/v4/chat/completions`

**请求头**：

```
Content-Type: application/json
Authorization: Bearer YOUR_API_KEY
```

**请求体**（翻译用途的 Prompt 工程）：

```json
{
    "model": "glm-4-flash",
    "messages": [
        {
            "role": "system",
            "content": "You are a professional translator. Translate the following text to {target_language}. Only output the translation result, no explanations."
        },
        {
            "role": "user",
            "content": "{text_to_translate}"
        }
    ],
    "temperature": 0.3,
    "max_tokens": 1024
}
```

**响应体**：

```json
{
    "choices": [
        {
            "message": {
                "role": "assistant",
                "content": "翻译结果文本"
            }
        }
    ]
}
```

#### 鉴权方式

**方式一：API Key 直接鉴权（推荐，简单）**

```
Authorization: Bearer <your-api-key>
```

**方式二：JWT Token 鉴权（更高安全性）**

```python
import time, jwt

def generate_token(apikey: str, exp_seconds: int):
    id, secret = apikey.split(".")
    payload = {
        "api_key": id,
        "exp": int(round(time.time() * 1000)) + exp_seconds * 1000,
        "timestamp": int(round(time.time() * 1000)),
    }
    return jwt.encode(payload, secret, algorithm="HS256",
                      headers={"alg": "HS256", "sign_type": "SIGN"})
```

> 客户端实现建议：采用 API Key 直接鉴权方式，与现有 LibreTranslate 的 `tc_translate_key` 配置复用，降低实现复杂度。

#### 相比现有后端的优势

| 对比维度   | 腾讯云 TMT | LibreTranslate | FTAPI  | 智谱AI              |
| ------ | ------- | -------------- | ------ | ----------------- |
| 费用     | 付费      | 免费自部署          | 免费（有限） | **免费额度**          |
| 注册门槛   | 腾讯云实名认证 | 需自建服务器         | 无需注册   | 手机号注册             |
| 翻译质量   | ★★★★    | ★★★            | ★★     | ★★★★★             |
| 上下文理解  | 无       | 无              | 无      | **有（LLM）**        |
| 自动翻译   | ✅       | ✅              | ❌      | ✅                 |
| 游戏术语适配 | 一般      | 一般             | 差      | **可通过 Prompt 优化** |
| 速率限制   | 按套餐     | 自定义            | 严格     | 免费额度内宽松           |

#### 实现要点

1. **继承** **`ITranslateBackendHttp`**：与现有后端保持一致的架构
2. **Prompt 模板**：通过 system prompt 指定翻译方向和风格，可针对游戏场景优化
3. **模型选择**：默认使用 `glm-4-flash`（免费），可通过 `qm_translate_zhipuai_model` 配置切换模型
4. **语言检测**：GLM 模型本身具备语言识别能力，可在 prompt 中要求返回源语言信息
5. **流式响应**：暂不支持（游戏聊天场景不需要逐字输出，等待完整结果即可）

## 四、数据流

### 当前数据流

```
聊天消息到达 (CChat::OnMessage)
    │
    ├─→ AutoTranslate() 检查是否需要自动翻译
    │       │
    │       └─→ Translate(Line, ShowProgress=false, AutoTriggered=true)
    │               │
    │               └─→ CreateTranslateBackend() 创建后端实例
    │                       │
    │                       └─→ HTTP 请求发出（异步）
    │
    ├─→ 用户手动执行 translate / translate_id 命令
    │       │
    │       └─→ Translate(Name/Id, ShowProgress=true)
    │               │
    │               └─→ 同上创建后端并发请求
    │
    └─→ OnRender() 每帧轮询
            │
            ├─→ 入站任务完成 → 更新 CLine.m_pTranslateResponse → RebuildChat()
            └─→ 出站任务完成 → SendChatQueued() 发送翻译后的消息
```

## 五、翻译任务类型

### 入站翻译（CTranslateJob）

翻译收到的聊天消息，结果存储在 `CChat::CLine::m_pTranslateResponse`（`std::shared_ptr<CTranslateResponse>`）中。

**当前支持**：其他语言 → 中文（目标语言默认 `zh`）

### 出站翻译（COutgoingTranslateJob）

翻译即将发送的消息。用户在消息末尾添加 `[ru]`、`[en]`、`[ja]` 等语言代码指定目标语言。

**当前状态**：出站翻译的发送逻辑在 chat.cpp 中**已被注释掉**：

```cpp
// TODO: Outgoing chat translation not implemented
// if(AllowOutgoingTranslation && GameClient()->m_Translate.TryTranslateOutgoingChat(Team, pLine))
// 	return;
```

> ⚠️ `TryTranslateOutgoingChat()` 的解析逻辑已完整实现，`OnRender()` 中出站任务完成的处理也已实现（调用 `SendChatQueued` 发送翻译结果），仅差 `SendChatQueued()` 入口处的调用被注释。

## 六、自动翻译逻辑

自动翻译在 translate.cpp 的 `AutoTranslate()` 中实现，规则如下：

1. 需要启用 `tc_translate_auto` 配置（默认启用）
2. 跳过客户端自身消息（`CLIENT_MSG`）
3. 跳过服务器消息（`SERVER_MSG`）
4. 跳过本地玩家消息
5. FTAPI 后端强制禁用自动翻译
6. **自动触发的翻译如果检测到中文源语言，会清空翻译结果**（跳过中文消息，因为目标语言默认为 `zh`）

## 七、🆕 聊天框翻译按钮

### 功能描述

在聊天框中为每条消息添加可点击的翻译按钮，使鼠标操作聊天框时可以一键翻译消息。这是对当前仅支持控制台命令翻译的重要交互升级。

### 交互设计

#### 入站翻译按钮（其他语言 → zh）

```
┌──────────────────────────────────────────────────────┐
│ [Player]: Hello, how are you?              [🌐 翻译] │  ← 鼠标悬停显示翻译按钮
│   ↳ 你好，你好吗？ [en]                               │  ← 点击后显示翻译结果
└──────────────────────────────────────────────────────┘
```

- **触发条件**：鼠标悬停在聊天消息行上时，右侧显示翻译图标按钮
- **点击行为**：触发该行的 `Translate(Line)` 翻译操作
- **状态反馈**：
  - 未翻译：显示 🌐 图标
  - 翻译中：显示 ⏳ 加载动画
  - 翻译完成：图标变为 ✅，下方显示翻译结果
  - 翻译失败：图标变为 ❌

#### 出站翻译按钮（zh → 其他语言）

```
┌──────────────────────────────────────────────────────┐
│ 输入框: 大家好                              [🌐 发送] │  ← 输入框旁的翻译发送按钮
│                                                      │
│ 点击后弹出目标语言选择：                                │
│   [en] English  [ja] 日本語  [ru] Русский  [更多...] │
│                                                      │
│ 选择后 → 翻译 → 自动发送翻译结果                       │
└──────────────────────────────────────────────────────┘
```

- **触发方式**：聊天输入框旁的翻译发送按钮
- **点击行为**：
  1. 弹出目标语言选择面板（常用语言 + 自定义输入）
  2. 翻译输入框中的文本
  3. 翻译完成后自动发送
- **快捷操作**：也可继续使用 `[en]`、`[ja]` 等消息末尾标记

### 实现要点

1. **聊天框鼠标交互**：当前聊天框已支持鼠标操作（`CChat` 中有鼠标事件处理），需扩展每行的悬停检测和按钮渲染
2. **翻译按钮渲染**：在 `CChat::OnRender()` 的消息行渲染逻辑中，检测鼠标悬停状态，在行尾绘制翻译图标
3. **点击事件处理**：在 `CChat::OnInput()` 中处理翻译按钮的点击事件，调用 `CTranslate::Translate(Line)`
4. **出站翻译面板**：新增一个小的语言选择 UI 弹出面板，可复用现有的 UI 组件模式
5. **配置项**：新增 `qm_translate_show_button` 配置项控制是否显示翻译按钮

### 代码结构

```cpp
class CTranslateBackendZhipuAI : public ITranslateBackendHttp
{
private:
    bool ParseResponseJson(const json_value *pObj, CTranslateResponse &Out)
    {
        // 解析智谱AI响应格式：
        // { "choices": [{ "message": { "content": "翻译结果" } }] }
        // 提取 choices[0].message.content 作为翻译文本
    }

protected:
    bool ParseResponse(CTranslateResponse &Out) override
    {
        json_value *pObj = m_pHttpRequest->ResultJson();
        bool Res = ParseResponseJson(pObj, Out);
        json_value_free(pObj);
        return Res;
    }

public:
    const char *Name() const override { return "ZhipuAI"; }

    CTranslateBackendZhipuAI(IHttp &Http, const char *pText, const char *pTarget)
    {
        // 1. 构造翻译 Prompt
        // 2. 构造 JSON 请求体（model, messages, temperature, max_tokens）
        // 3. 设置 Authorization: Bearer <api_key> 请求头
        // 4. 发送 POST 请求到智谱AI端点
    }
};
```
