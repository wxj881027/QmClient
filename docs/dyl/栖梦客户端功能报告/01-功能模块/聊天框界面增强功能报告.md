## 三、当前问题分析

### 3.1 消息区域未使用 QmUi 系统

聊天框消息区域使用 chat.cpp 内置的手动动画系统，**未接入 QmUi 的 LayoutEngine 和 CUiV2Tree**。只有聊天气泡（玩家头顶）使用了 `CUiV2AnimationRuntime`。

这导致：
- 动画效果有限，无法利用 QmUi 的高级动画能力（弹簧动画、序列动画等）
- 布局计算手动硬编码，难以实现响应式设计
- 与其他使用 QmUi 的组件（聊天气泡、HUD 灵动岛等）风格不统一

### 3.2 输入框视觉简陋

当前输入框只是简单文本渲染 + 前缀，缺少：
- 输入框背景/边框
- 光标动画
- 输入焦点指示
- 字符计数/限制提示

### 3.3 消息过渡不够丝滑

- 新消息出现时只有简单的滑入+淡入
- 消息被挤出时只有 alpha 裁剪，没有平滑的位移动画
- 消息之间没有间距动画

### 3.4 缺少交互反馈

- 没有消息悬停效果
- 没有消息点击交互（如复制、翻译按钮）
- 没有输入状态指示（如"正在输入..."）

## 四、增强方案

### 4.1 迁移到 QmUi 动画系统（高优先级）

将消息区域的动画从手动计算迁移到 `CUiV2AnimationRuntime`：

**当前**：
```cpp
float AnimAlpha = CalculateAnimationAlpha(MessageAge, m_PrevShowChat);
float AnimOffsetX = CalculateAnimationOffsetX(MessageAge, Emphasized, m_PrevShowChat);
```

**迁移后**：
```cpp
// 为每条消息创建 QmUi 动画节点
CUiV2AnimationRuntime *pAnim = GameClient()->UiRuntimeV2()->AnimRuntime();
pAnim->SetValue(Line.m_AnimNodeKey, EUiAnimProperty::ALPHA, 0.0f);
pAnim->RequestAnimation({Line.m_AnimNodeKey, EUiAnimProperty::ALPHA, 1.0f, 0.3f, EEaseType::EASE_OUT});
pAnim->SetValue(Line.m_AnimNodeKey, EUiAnimProperty::POS_X, -25.0f);
pAnim->RequestAnimation({Line.m_AnimNodeKey, EUiAnimProperty::POS_X, 0.0f, 0.3f, EEaseType::EASE_OUT_BACK});
```

**优势**：
- 统一动画系统，与聊天气泡、HUD 灵动岛风格一致
- 可使用弹簧动画、序列动画等高级效果
- 动画参数可配置化

### 4.2 输入框现代化（中优先级）

```
┌─────────────────────────────────────────────┐
│  All: │ 你好世界_                        │  ← 光标闪烁
│       │ [Tab 补全提示]                    │
└─────────────────────────────────────────────┘
```

增强内容：
- 输入框背景（半透明圆角矩形）
- 光标闪烁动画（0.5s 闪烁周期）
- 输入焦点边框（高亮色）
- 字符计数器（接近上限时显示）

### 4.3 消息过渡动画增强（中优先级）

| 效果 | 当前 | 增强 |
|------|------|------|
| 新消息出现 | 滑入+淡入 | 弹簧滑入+缩放弹出+淡入 |
| 消息淡出 | alpha 衰减 | 向左滑出+缩小+淡出 |
| 消息被挤出 | alpha 裁剪 | 平滑向上位移+淡出 |
| 高亮消息 | 更大滑入偏移 | 弹簧效果+短暂放大+发光边框 |

### 4.4 消息间距动画（低优先级）

当新消息插入时，旧消息平滑向下推移，而非瞬间跳变：

```
旧消息 A ──→ 平滑下移
旧消息 B ──→ 平滑下移
新消息 C ──→ 弹簧滑入
```

### 4.5 配置变量

| 变量 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `qm_chat_anim_style` | INT | 1 | 动画风格（0=经典, 1=现代, 2=弹簧） |
| `qm_chat_input_bg` | INT | 1 | 输入框背景 |
| `qm_chat_cursor_blink` | INT | 1 | 光标闪烁 |
| `qm_chat_msg_gap` | INT | 2 | 消息间距（0-10像素） |

## 五、实现路径

### 阶段一：QmUi 动画迁移

1. 为每条消息分配 QmUi 动画节点 Key
2. 将 `CalculateAnimationAlpha/OffsetX` 替换为 `CUiV2AnimationRuntime` 调用
3. 保持视觉行为不变，仅替换底层实现

**修改文件**：

| 文件 | 修改内容 |
|------|----------|
| `chat.h` | 添加 `m_AnimNodeKey` 字段到 `CLine` |
| `chat.cpp` | 替换动画计算逻辑 |

### 阶段二：输入框增强

1. 添加输入框背景渲染
2. 实现光标闪烁动画
3. 添加焦点边框

### 阶段三：高级动画效果

1. 弹簧动画效果
2. 消息间距平滑过渡
3. 高亮消息发光效果
