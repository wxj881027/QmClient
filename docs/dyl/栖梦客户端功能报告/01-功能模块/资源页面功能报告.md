# QmClient 资源页面功能报告

## 一、总体架构概览

QmClient 的资源管理功能分布在**设置菜单**的多个页面中，没有统一的"资源页面"入口。当前资源管理呈现**分散式**结构：皮肤（Skins）在 "Tee" 页面，其他贴图资源（Entities/Game/Emoticons/Particles/HUD/Extras）在 "Assets" 页面，且两者使用完全不同的代码架构和交互模式。

---

## 三、皮肤浏览页面（Tee 页面）

**位置**: `SETTINGS_TEE` -> `RenderSettingsTee()`

**主要文件**: `menus_settings.cpp:415-1300+`、`skins.h`、`skins.cpp`

**核心类**:

| 类名 | 文件 | 功能 |
|------|------|------|
| `CSkins` | skins.h | 皮肤管理核心组件，负责加载、下载、缓存 |
| `CSkins::CSkinContainer` | skins.h:65 | 单个皮肤的容器，管理加载状态 |
| `CSkins::CSkinListEntry` | skins.h:165 | 皮肤列表条目（含收藏/选中状态） |
| `CSkins::CSkinList` | skins.h:206 | 皮肤列表（含搜索过滤） |
| `CSkins::CSkinDownloadJob` | skins.h:327 | 皮肤下载异步任务 |

**皮肤类型** (`CSkinContainer::EType`):
- `LOCAL` - 本地皮肤
- `DOWNLOAD` - 在线下载的皮肤

**皮肤下载机制**:
- 配置项 `cl_download_skins` 控制是否启用自动下载
- 下载 URL: `cl_skin_download_url` (默认 `https://skins.ddnet.org/skin/`)
- 社区皮肤 URL: `cl_skin_community_download_url`
- 下载的皮肤保存到 `downloadedskins/` 目录

**皮肤浏览功能**:
- 网格列表展示，每行显示 Tee 预览 + 皮肤名称
- 搜索过滤 (`cl_skin_filter_string`)
- 收藏功能（`AddFavorite`/`RemoveFavorite`）
- 皮肤队列（Skin Queue）系统，支持预设（Presets）
- "Skin Database" 按钮链接到 `https://ddnet.org/skins/`
- "Skins directory" 按钮打开本地皮肤目录
- 刷新按钮 (F5)

**Tee 页面子标签**: Player / Dummy / Profiles 三个子标签

## 五、Workshop 在线下载系统

**这是 QmClient 的核心特色功能**，允许用户在线浏览和下载社区贡献的贴图资源。

**数据结构**:

- `SWorkshopHudAsset` — 单个 Workshop 资源条目
  - `m_Id`, `m_Name`, `m_Author` - 资源元数据
  - `m_ImageUrl` - 资源图片 URL
  - `m_ThumbCachePath` - 缩略图缓存路径
  - `m_InstallPath` - 安装路径
  - `m_ThumbTexture` - 缩略图 GPU 纹理
  - `m_pThumbTask` / `m_pDownloadTask` - 异步 HTTP 任务
  - `m_Installed` - 是否已安装
  - `m_DownloadFailed` - 下载是否失败

- `SWorkshopHudState` — Workshop 状态管理
  - `m_vAssets` - 资源列表
  - `m_pListTask` - 列表获取 HTTP 任务
  - `m_Requested` / `m_LoadFailed` - 请求状态
  - `m_CacheTime` / `m_LastRefreshTime` - 缓存时间

**Workshop 状态实例** (每种资源类型独立):
```cpp
static SWorkshopHudState gs_WorkshopHudState;
static SWorkshopHudState gs_WorkshopEntitiesState;
static SWorkshopHudState gs_WorkshopGameState;
static SWorkshopHudState gs_WorkshopEmoticonsState;
static SWorkshopHudState gs_WorkshopParticlesState;
```

**注意**: `ASSETS_TAB_EXTRAS` 没有 Workshop 支持。

**API 端点**: `https://www.ddrace.cn/data/assets.json`

通过 `category` 字段区分资源类型: `"实体层"`、`"游戏"`、`"表情"`、`"粒子"`、`"HUD"`

**下载流程**:
1. 首次加载时尝试从本地缓存 (`qmclient/workshop/`) 读取
2. 缓存不存在则发起 HTTP 请求获取资源列表
3. 解析 JSON 响应，按 category 过滤
4. 增量更新：合并新旧数据，保留已有纹理和任务
5. 缩略图异步下载 + 解码 (`CImageDecodeJob`)
6. 点击下载时，弹出确认框，确认后通过 `HttpGetFile` 下载到 `assets/<type>/` 目录
7. 支持删除已安装资源（带确认弹窗）

**缓存机制**:
- 缩略图缓存: `qmclient/workshop/thumbs/<id>.webp`
- 列表缓存: `qmclient/workshop/workshop_<type>.json`

## 六、贴图编辑器（Assets Editor）

**入口**: Assets 页面底部的 "Assets editor" 按钮

**编辑器类型**:
- `ASSETS_EDITOR_TYPE_GAME` - 游戏贴图
- `ASSETS_EDITOR_TYPE_EMOTICONS` - 表情贴图
- `ASSETS_EDITOR_TYPE_ENTITIES` - 实体层
- `ASSETS_EDITOR_TYPE_HUD` - HUD 贴图
- `ASSETS_EDITOR_TYPE_PARTICLES` - 粒子贴图
- `ASSETS_EDITOR_TYPE_EXTRAS` - 额外贴图

功能包括：从现有资源拖拽组合、预览、导出为自定义贴图文件。

## 七、方向键资源

方向箭头在 `menus.h:85` 中有 `m_DirectionQuadContainerIndex` 成员，用于在玩家头顶渲染方向箭头。这不是一个独立的资源浏览页面，而是游戏内渲染功能的一部分。

## 八、当前组织方式总结

| 资源类型 | 所在页面 | 浏览方式 | 在线下载 | 本地管理 |
|----------|----------|----------|----------|----------|
| **皮肤 (Skins)** | Tee 页面 | 网格列表 + Tee 预览 | 自动下载 (DDNet 皮肤服务器) | 收藏/队列/预设 |
| **实体层 (Entities)** | Assets 页面 | 列表 + 纹理预览 | Workshop (ddrace.cn) | 安装/删除 |
| **游戏 (Game)** | Assets 页面 | 列表 + 纹理预览 | Workshop (ddrace.cn) | 安装/删除 |
| **表情 (Emoticons)** | Assets 页面 | 列表 + 纹理预览 | Workshop (ddrace.cn) | 安装/删除 |
| **粒子 (Particles)** | Assets 页面 | 列表 + 纹理预览 | Workshop (ddrace.cn) | 安装/删除 |
| **HUD** | Assets 页面 | 列表 + 纹理预览 | Workshop (ddrace.cn) | 安装/删除 |
| **额外 (Extras)** | Assets 页面 | 列表 + 纹理预览 | 无 | 安装/删除 |
| **方向箭头** | 无独立页面 | 游戏内渲染 | 无 | 无 |

## 九、关键发现与问题

1. **皮肤与贴图资源分离**: 皮肤在 "Tee" 页面，其他贴图在 "Assets" 页面，两者使用完全不同的代码架构。皮肤有独立的 `CSkins` 组件和复杂的队列/预设系统，而贴图资源使用 `SCustomItem` 系列结构体 + Workshop 下载。

2. **Workshop 不覆盖所有类型**: `ASSETS_TAB_EXTRAS` 没有在线 Workshop 支持，只有本地资源浏览。

3. **皮肤无 Workshop**: 皮肤下载使用 DDNet 官方皮肤服务器（按需下载），没有像贴图资源那样的"在线商店"浏览体验。只有一个 "Skin Database" 外部链接按钮。

4. **代码重复严重**: `menus_settings_assets.cpp` 中大量 if-else 链根据 `s_CurCustomTab` 分支处理不同资源类型，逻辑高度相似但无法统一。

5. **Workshop 状态管理**: 5 个独立的 `SWorkshopHudState` 静态变量，每个都需要单独管理，增加了维护复杂度。

6. **贴图编辑器**: Assets Editor 是一个功能完整的组合式编辑器，但入口较深。

7. **方向键资源**: 方向箭头目前没有独立的资源浏览页面，无法自定义或更换。
