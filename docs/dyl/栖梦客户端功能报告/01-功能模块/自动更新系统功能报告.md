### 2.4 更新流程

```
InitiateUpdate()
    │
    ├─→ 直接下载 DDNet.exe（最新 Release）
    │
    └─→ PerformUpdate()
         │
         ├─→ 解析 update/update.json 清单
         │    - JSON 数组，每个版本条目含 version/client/server/download/remove 字段
         │    - 遇到 version == TCLIENT_VERSION 时停止（只处理更新版本）
         │
         └─→ RunningUpdate()
              │
              ├─→ 逐个下载文件（支持平台特定 DLL/SO）
              │
              └─→ CommitUpdate()
                   │
                   ├─→ 移动文件到最终位置
                   ├─→ ReplaceClient()：重命名旧文件为 .old，移入新文件
                   └─→ 状态设为 NEED_RESTART
```

### 2.5 版本比较逻辑

[ParseUpdate()](file:///e:/Coding/DDNet/QmClient/src/engine/client/updater.cpp#L290-L362)：

- 读取 `update/update.json`（JSON 数组）
- 遍历每个版本条目
- 当遇到 `version` 字段等于 `TCLIENT_VERSION` 时停止
- 之前的所有条目都是需要更新的版本
- 每个版本条目包含 `client`/`server` 布尔值和 `download`/`remove` 文件列表

### 2.6 可执行文件替换

[ReplaceClient()](file:///e:/Coding/DDNet/QmClient/src/engine/client/updater.cpp#L240-L263)：

- Windows：重命名旧文件为 `.old`，将新文件从 `update/` 移入
- Linux：额外执行 `chmod +x`

## 五、当前问题分析

### 5.1 没有"检查更新"功能

当前 `InitiateUpdate()` 直接开始下载，没有先检查是否有新版本。用户无法：
- 知道当前是否是最新版本
- 查看新版本的更新日志
- 选择是否下载更新

### 5.2 没有版本号对比提示

更新器通过解析 `update.json` 清单来判断是否需要更新，但不会向用户显示"当前版本 X → 新版本 Y"的信息。

### 5.3 没有更新日志展示

即使 GitHub Releases 有详细的 changelog，更新器也不会获取和展示。

### 5.4 没有 GitHub API 集成

当前直接下载 `DDNet.exe`，未使用 GitHub API 查询最新 Release 信息（版本号、changelog、下载 URL）。

### 5.5 更新源 URL 可能过时

当前 URL 指向 `wxj881027/Q1menG_Client`，但文档网站使用的是 `sjrc6/TaterClient-ddnet`，两者不一致。

### 5.6 缺少更新设置 UI

用户无法在设置中：
- 开关自动更新
- 设置更新检查频率
- 查看当前版本信息

## 六、增强方案

### 6.1 GitHub Releases API 集成（高优先级）

使用 GitHub API 获取最新 Release 信息：

```
GET https://api.github.com/repos/{owner}/{repo}/releases/latest
```

响应包含：
- `tag_name`：版本标签
- `name`：Release 标题
- `body`：更新日志（Markdown）
- `published_at`：发布时间
- `assets[]`：下载文件列表（含 `browser_download_url`、`size`、`download_count`）

#### 实现方案

新增 `CCheckUpdateJob` 类，继承 `CHttpRequest`：

```cpp
class CCheckUpdateJob : public CHttpRequest
{
    SUpdateInfo m_Info;  // 版本号、changelog、下载 URL

    void OnCompletion(EHttpState State) override
    {
        if(State == EHttpState::DONE)
        {
            // 解析 GitHub API JSON 响应
            // 填充 m_Info
        }
    }
};
```

### 6.2 版本比较逻辑

使用语义化版本比较（SemVer）：

```
当前版本: TCLIENT_VERSION = "2.35.0"
最新版本: GitHub Release tag = "v2.36.0"

比较: 2.35.0 < 2.36.0 → 有更新
```

### 6.3 更新通知 UI

启动时后台检查更新，如果有新版本：

```
┌──────────────────────────────────────────────┐
│  🔄 发现新版本: v2.36.0                       │
│  当前版本: v2.35.0                            │
│                                               │
│  更新内容:                                    │
│  - 新增智谱AI翻译后端                         │
│  - 修复聊天框闪退问题                         │
│  - 优化渲染性能                               │
│                                               │
│  [立即更新]  [稍后提醒]  [跳过此版本]         │
└──────────────────────────────────────────────┘
```

### 6.4 配置变量

| 变量 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `qm_update_check` | INT | 1 | 启动时自动检查更新 |
| `qm_update_check_interval` | INT | 86400 | 检查间隔（秒），默认每天 |
| `qm_update_skip_version` | STR(32) | `""` | 跳过的版本号 |
| `qm_update_repo` | STR(128) | `"wxj881027/Q1menG_Client"` | GitHub 仓库 |

### 6.5 更新流程增强

```
客户端启动
    │
    ├─→ 检查 qm_update_check 是否启用
    │       │
    │       └─→ 检查上次检查时间是否超过间隔
    │               │
    │               └─→ 后台请求 GitHub API
    │                       │
    │                       ├─→ 有新版本且未跳过 → 显示更新通知
    │                       └─→ 已是最新 → 静默
    │
    └─→ 用户点击"立即更新"
            │
            ├─→ 下载最新 Release 资产
            ├─→ 验证 SHA256（如果 Release 提供）
            ├─→ 替换文件
            └─→ 提示重启
```
