## 附录：Opus DLL 修复说明

### 问题描述
在 Windows 上运行 `cargo test` 时，测试因缺少 `libopus.dll` 而失败：
```
error: process didn't exit successfully: STATUS_DLL_NOT_FOUND
```

### 修复步骤

#### 1. 运行设置脚本
```powershell
cd e:\Coding\DDNet\QmClient
pwsh -File qmclient_scripts\setup-opus.ps1
```

脚本会自动完成：
- ✅ 从 `libopus.dll` 提取导出符号 (67 个 opus_* 函数)
- ✅ 生成 MSVC 导入库 `opus.lib`
- ✅ 复制运行时 DLL (`libopus.dll`, `libwinpthread-1.dll`) 到 `target/debug/deps/`

#### 2. 文件位置
| 文件 | 路径 |
|------|------|
| 原始 DLL | `ddnet-libs/opus/windows/lib64/libopus.dll` |
| MSVC 导入库 | `src/rust-bridge/voice/opus-prebuilt/lib/opus.lib` |
| 运行时 DLL | `target/debug/deps/libopus.dll` |

#### 3. 验证修复
```powershell
cd src/rust-bridge/voice
cargo test codec::opus
```

**预期结果**: 所有 Opus 测试通过
```
running 9 tests
test codec::opus::tests::test_adapt_to_network ... ok
test codec::opus::tests::test_bitrate_change ... ok
test codec::opus::tests::test_codec_creation ... ok
...
test result: ok. 9 passed; 0 failed
```

### 技术细节
- **原因**: `audiopus_sys` crate 需要链接 Opus 库，ddnet-libs 提供的 `libopus.dll` 是 MinGW 格式
- **解决**: 脚本使用 `dumpbin` 提取导出符号，使用 `lib.exe` 生成 MSVC 兼容的导入库
- **副作用**: 无，生成文件已加入 `.gitignore`
