### 1.2 发现的问题

#### 🔴 CRITICAL: 无

#### 🟠 HIGH: 1 个问题

**问题 H1**: `estimate_echo()` 和 `update_nlms()` 中的 Vec 分配  
**位置**: `src/dsp/aec.rs:166`, `src/dsp/aec.rs:193`  
**问题描述**: 
```rust
// 当前实现（每次调用都分配 Vec）
let far_slice: Vec<_> = self.far_end_buffer.iter().copied().collect();
```
这在热路径（每帧调用）中会造成堆分配，违反性能要求。  
**建议修复**: 使用迭代器直接访问，避免 Vec 分配：
```rust
// 修复后
let buffer_len = self.far_end_buffer.len();
self.weights.iter().enumerate().take(buffer_len).map(|(i, &weight)| {
    let idx = buffer_len - 1 - i;
    weight * self.far_end_buffer[idx]
}).sum()
```

#### 🟡 MEDIUM: 2 个问题

**问题 M1**: `CaptureQueue` 使用 Mutex 而非无锁队列  
**位置**: `src/worker/context.rs:29`  
**问题描述**: 
```rust
pub struct CaptureQueue {
    buffer: Mutex<Vec<AudioFrameEntry>>,  // 使用了 Mutex
    // ...
}
```
虽然使用了原子变量管理读写位置，但 `buffer` 字段仍使用 `Mutex`，在音频捕获线程中可能成为瓶颈。  
**建议**: 考虑使用 `crossbeam::ArrayQueue` 实现真正的无锁队列。

**问题 M2**: 部分 FFI 函数缺少参数验证  
**位置**: `src/lib.rs` 多个 FFI 函数  
**问题描述**: 某些 FFI 函数没有验证输入指针是否为 null：
```rust
ffi_wrap!(unsafe fn voice_config_from_cabi(cfg: *const VoiceConfigCABI) -> u32 {
    // 缺少 cfg.is_null() 检查
    let cfg = &*cfg;
    // ...
})
```
**建议修复**: 添加 null 指针检查：
```rust
ffi_wrap!(unsafe fn voice_config_from_cabi(cfg: *const VoiceConfigCABI) -> u32 {
    if cfg.is_null() {
        log::error!("voice_config_from_cabi: null pointer");
        return 0;
    }
    // ...
})
```

**问题 H2**: `AudioMixer::mix()` 返回 Vec 造成分配  
**位置**: `src/audio/mixer.rs`  
**问题描述**: 混音函数返回 `Vec<i16>`，调用者需要频繁分配内存。

pub fn mix_into(&self, output: &mut [i16], samples: usize, channels: usize) -> usize

**问题 M3**: `PlaybackBuffer` 读取逻辑复杂  
**位置**: `src/worker/context.rs:218-243`  
**问题描述**: 环形缓冲区读取逻辑使用取模运算，可能影响缓存效率。  
**建议**: 考虑使用双缓冲或简化索引计算。

**问题 M4**: `DspChain` 缺少处理延迟统计  
**位置**: `src/dsp/mod.rs`  
**问题描述**: 没有测量和报告 DSP 处理时间，难以监控实时性能。  
**建议**: 添加处理时间统计，超过阈值时记录警告。

### 3.2 发现的问题

#### 🟠 HIGH: 1 个问题

**问题 H3**: `VoicePacket::parse()` 缺少边界检查  
**位置**: `src/network/protocol.rs`  
**问题描述**: 解析函数没有充分验证数据长度，可能导致缓冲区溢出：
```rust
pub fn parse(data: &[u8]) -> Result<Self, ProtocolError> {
    // 检查 magic 和 version
    if data.len() < 6 {
        return Err(ProtocolError::BufferTooSmall);
    }
    // ... 后续解析可能越界
}
```
**建议**: 在每个解析步骤前验证剩余缓冲区大小。

#### 🟡 MEDIUM: 2 个问题

**问题 M5**: 抖动缓冲区大小固定  
**位置**: `src/lib.rs:105`  
**问题描述**: `MAX_JITTER_PACKETS` 固定为 32，不能适应不同网络条件。  
**建议**: 根据网络延迟动态调整抖动缓冲区大小。

**问题 M6**: 缺少序列号回绕处理  
**位置**: `src/jitter/mod.rs`  
**问题描述**: u16 序列号在 65535 后回绕到 0，可能导致包顺序判断错误。  
**建议**: 使用 RFC 1982 序列号比较算法。
