# QmClient 功能开发计划

## 文件说明

### 结构分类

| 分类 | 说明 |
|------|------|
| **一、UI/UX 增强** | 界面美化、动画效果、交互体验优化 |
| **二、网络与性能** | 网络诊断、延迟优化、卡顿修复、协议改进 |
| **三、社交与社区** | 好友系统、聊天、服务器收藏、社交功能 |
| **四、资源与内容** | 皮肤、地图、截图、Workshop 资源管理 |
| **五、游戏功能** | 翻译、Bind指令、Demo播放器、观战模式、统计 |
| **六、系统与工具** | 配置导入导出、汉化、自动更新、语音、插件系统 |
| **七、功能移植** | 从BC端移植的功能（3D粒子、计时器、媒体背景等） |
| **八、持续集成** | CI/CD、自动化构建相关 |
| **九、已排除的创意** | 经评估决定不开发的功能 |

### 优先级标记

- **（高优先级）** — 建议优先实现，影响面广或基础功能
- **（调研）** — 需要前期调研评估可行性
- **加粗** — 特别推荐的功能

### 状态标记

- `- [ ]` — 待完成
- `- [x] ~~内容~~` — 已完成（使用删除线标记）

### 编写规范

1. **分类层级**：使用 `##` 主分类 → `###` 子分类 → `- [ ]` 任务项
2. **子任务缩进**：使用2个空格缩进子任务或补充说明
3. **技术细节**：在任务项下方补充实现思路、相关代码文件、依赖关系
4. **已完成项**：移动到对应分类，使用 `- [x] ~~内容~~` 格式

## 一、UI/UX 增强

### 1. 界面现代化
- [ ] UI 组件合并
- [ ] 动画和UI风格统一
  - 合并三套重复动画辅助函数（`ResolveAnimatedLayoutValueEx`/`ResolveUiAnimValue`/`ResolveAnimatedValue`）
  - 旧版 `CUiEffects` 逐步迁移到 V2 `CUiV2AnimationRuntime`
- [ ] 统一 UI 主题/样式系统
  - 当前颜色/样式分散在数十个配置变量中，`SUiStyle` 仅覆盖布局不含视觉属性
  - 需要集中管理的 Theme/Style 系统（颜色、圆角、阴影等）

### 2. 品牌化迁移（Qimeng → QmClient）
- [ ] 高优先级：`lyrics_component.cpp` 中 User-Agent `"QimenGClient"` → `"QmClient"`
- [ ] 中优先级：`ConfigDomain::QIMENG`、`config_variables_qimeng.h`、`SETTINGS_QIMENG`、`RenderSettingsQiMeng` 等代码标识符
- [ ] 低优先级/需谨慎：`CTClient` 类名和 `tc_` 配置前缀（100+ 处引用，影响配置文件兼容性）

### 3. 聊天框界面增强
- [ ] 迁移消息区域动画到 QmUi `CUiV2AnimationRuntime`（当前仅聊天气泡使用了 QmUi，消息区域用手动动画）

### 4. 聊天框翻译按钮

### 5. 钩子反馈时间进度条
- [ ] 为钩子技能添加时间进度条，显示在钩子释放者角色旁边
  - 钩子时长由 `HookDuration` 调谐参数控制（默认 1.25 秒），钩住玩家最大 1.2 秒
  - 客户端已有 `m_HookTick`、`m_HookState`、`m_EndlessHook` 数据
  - 名字板系统已有 `CNamePlatePartHookStrongWeak` 组件可扩展
- [ ] 无限钩时进度条替换为 ♾️ 符号
  - 无限钩通过 `CHARACTERFLAG_ENDLESS_HOOK` 标志位网络同步

- [ ] 进度条颜色渐变（绿→黄→红），低时间闪烁

### 6. 武器切换甩枪动画

### 7. HUD 编辑器完善（调研）
- [ ] 已有 HudEditor 基础功能，需调研当前完善程度和可改进方向

### 8. 无障碍功能

## 二、网络与性能

### 1. 网络卡顿诊断
- [ ] 显示服务器卡顿还是客户端卡顿
  - 分离上行/下行延迟显示（引擎注释已有 Snapshot Latency=下行、Prediction Latency=上行概念，未暴露给 UI）
  - 丢包率 + Jitter 显示（`NETSTATS` 只有收发包计数；引擎内部有 `m_AutoMarginLatencyJitterMs` 未暴露）
  - 连接质量分级：当前 `ConnectionProblems()` 只有"有/无"二值，应改为多级（正常/偏高/严重/断开）
- [ ] NetGraph 网络延迟历史图表（引擎已有 `m_FpsGraph`，但没有对应的网络延迟图表）
- [ ] 性能监控面板（联动 QmClient 状态栏 `tc_statusbar`，添加 FPS/CPU/GPU/内存/Draw Call 等指标项）
- [ ] 能解决游戏卡顿问题最好

### 2. 自适应 AntiPing
- [ ] 配置变量丰富（`cl_antiping_*` + `tc_antiping_*`），但缺乏自动调节逻辑
- [ ] 根据实时 Ping 自动调整 `cl_antiping_percent` 和 `cl_antiping_limit`
- [ ] 网络状态 HUD 和优化建议

### 3. 页面切换卡顿与崩溃优化
- [ ] 调研页面切换时的性能瓶颈
- [ ] 优化资源加载和渲染逻辑
- [ ] 减少崩溃问题

### 4. 网络协议改进与 Rust 应用
- [ ] 评估 Rust 在网络模块的应用场景
- [ ] 提升网络协议的安全性和性能

## 六、系统与工具

### 1. 配置导入/导出系统
- [ ] 当前配置分散在多个文件中，需要统一打包方案：
  - `settings_ddnet.cfg` — DDNet 原版设置
  - `settings_qmclient.cfg` — 栖梦客户端设置
  - `qmclient_warlist.cfg` — 黑名单
  - `qmclient_profiles.cfg` — 皮肤配置文件
  - `qmclient_chatbinds.cfg` — 聊天绑定
  - 用户自定义 cfg 文件（按键绑定、高级自定义操作，有文件夹）
- [ ] 实现方案：
  1. **打包为单一 `.qmconfig` 文件**：将所有 cfg 文件内容合并为一个 ZIP/JSON 归档，包含元数据（版本号、导出时间、客户端版本）
  2. **选择性导出**：用户可勾选要导出的配置类别（如只导出键位+翻译设置，不导出图形设置）
  3. **导入时智能合并**：检测冲突项，提供"覆盖/保留/逐项选择"三种模式
  4. **版本兼容**：导入时检查配置版本，旧版配置自动迁移
  5. **分享友好**：导出文件可分享给朋友，一键导入完整配置

### 2. 汉化完善
- [ ] 创建 QmClient 扩展语言文件 `data/qmclient/languages/simplified_chinese.txt`（双层加载机制第二层从未生效）
- [ ] 修复 ~117 处硬编码中文（A类 `Localize("中文")` 约50处 + B类 直接中文字符串约67处）
- [ ] 编辑器本地化（约121处硬编码中文，当前只有中文界面无法切换语言）
- [ ] 翻译文件维护：清理重复条目、按字母排序

### 3. 配置变量前缀统一
- [ ] `qmc_foot_particles` → `qm_foot_particles`（1个变量，低风险）
- [ ] `cl_scoreboard_*` → `qm_scoreboard_*`（2个变量，与上游 `cl_` 冲突）
- [ ] `Ri*`/`ri_*` → `QmVoice*`/`qm_voice_*`（约48个变量，C++成员名与脚本名不一致）
- [ ] 旧版聊天气泡 `tc_chat_bubble_*` 标记废弃（与 `qm_chat_bubble_*` 功能重复）

### 4. 自动更新系统

- [ ] 提示下载更新，避免用户手动去 GitHub 下载
