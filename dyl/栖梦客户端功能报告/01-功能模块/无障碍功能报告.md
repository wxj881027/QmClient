# QmClient 无障碍功能报告

## 一、功能概述

无障碍功能旨在让 QmClient 对色觉障碍、视力障碍和运动障碍用户更加友好。当前项目几乎没有无障碍相关实现，需要从零建立基础设施。

---

## 二、当前实现状态：几乎为零 ❌

搜索 `colorblind`、`accessibility`、`contrast` 等关键词，在整个代码库中**没有找到任何无障碍功能的实现或配置**。唯一的 `accessibility` 出现在 `access_level` 命令中（控制命令权限级别），与无障碍无关。

## 六、改进方案

### 6.1 色盲模式（高优先级）

#### 6.1.1 配置变量

| 变量 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `qm_colorblind_mode` | INT | 0 | 色盲模式（0=关, 1=红绿盲, 2=蓝黄盲, 3=全色盲） |

#### 6.1.2 色盲友好调色板

| 模式 | Ping 颜色方案 | 队伍颜色方案 | 状态颜色方案 |
|------|--------------|-------------|-------------|
| 正常 | 绿→红 HSL | 红/蓝 | 绿(好)/红(差) |
| 红绿盲 | 蓝→橙 HSL | 橙/蓝 | 蓝(好)/橙(差) |
| 蓝黄盲 | 红→绿 HSL | 红/绿 | 红(好)/绿(差) |
| 全色盲 | 亮→暗 灰度 | 亮灰/暗灰 | 亮(好)/暗(差) |

#### 6.1.3 实现方式

创建统一的颜色查询函数，替代所有硬编码颜色：

```cpp
ColorRGBA GetAccessibleColor(EColorContext Context, float Value);
```

所有颜色区分位置改为调用此函数，根据 `qm_colorblind_mode` 返回不同颜色。

#### 6.1.4 需要修改的位置

| 位置 | 当前颜色 | 改为调用 |
|------|----------|---------|
| 浏览器 Ping | HSL 硬编码 | `GetAccessibleColor(PING, latency)` |
| 计分板 Ping | HSL 硬编码 | `GetAccessibleColor(PING, latency)` |
| 名字板 Ping 圆环 | HSL 硬编码 | `GetAccessibleColor(PING, ping)` |
| 队伍颜色 | 硬编码红蓝 | `GetAccessibleColor(TEAM, team_id)` |
| HUD 状态 | 硬编码红绿 | `GetAccessibleColor(STATUS, value)` |
| Hook 强弱 | 硬编码红蓝白 | `GetAccessibleColor(HOOK, state)` |

### 6.2 全局 UI 缩放（中优先级）

| 变量 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `qm_ui_scale` | INT | 100 | 全局 UI 缩放百分比（50~200） |

实现方式：在 `CUiV2LegacyAdapter` 或渲染层添加全局缩放因子，所有 UI 元素自动按比例缩放。

### 6.3 高对比度模式（中优先级）

| 变量 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `qm_high_contrast` | INT | 0 | 高对比度模式 |

效果：
- 增强文字与背景的对比度
- 为半透明背景添加不透明边框
- 增大图标和文字的描边宽度

### 6.4 键盘焦点导航（低优先级）

实现可见焦点指示器（focus ring），支持 Tab 在 UI 元素间导航。这是最大的工程量，需要：

1. 焦点管理系统（当前焦点元素、Tab 序列）
2. 可见焦点指示器渲染
3. 所有 UI 组件支持焦点状态
4. 键盘快捷键增强

### 6.5 辅助标识（低优先级）

为所有颜色区分添加非颜色辅助标识：

| 功能 | 当前 | 增强 |
|------|------|------|
| Ping 值 | 仅颜色 | 添加 ↑↓ 箭头图标 |
| 队伍区分 | 仅颜色 | 添加队伍符号标记 |
| 状态指示 | 仅颜色 | 添加 ✓✗ 图标 |
| 游戏类型 | 仅颜色 | 添加文字缩写标签 |
