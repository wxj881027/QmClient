# QmClient 聊天框翻译按钮功能报告

## 一、聊天框鼠标可操作性 — 当前状态：完全不支持 ❌

### 当前实现

CChat 组件**没有任何鼠标交互功能**：

- **没有鼠标点击处理**：`OnInput()` 方法仅处理键盘输入（Escape、Enter、Tab自动补全、方向键历史记录），完全没有鼠标事件处理
- **没有** **`OnCursorMove`** **覆写**：CChat 没有覆写 `CComponent::OnCursorMove()`，因此无法追踪鼠标位置
- **没有 HotItem/DoButton 调用**：整个 chat.cpp 中唯一的 `Ui()` 调用是 `Ui()->DoSmoothScrollLogic()`，用于输入框滚动，没有任何按钮或悬停检测
- **没有聊天行悬停效果**：聊天消息渲染是纯输出性的，没有基于鼠标位置的视觉反馈

### 对比参考：记分板中的按钮

记分板组件有聊天静音按钮的实现，使用了 `Ui()->DoButton_FontIcon()` 模式：

```cpp
if(pUi->DoButton_FontIcon(&pPopupContext->m_MuteAction, FontIcons::FONT_ICON_BAN,
    Client.m_ChatIgnore, &Action, BUTTONFLAG_LEFT, ActionCorners))
{
    Client.m_ChatIgnore ^= 1;
}
```

这是聊天框中添加按钮可参考的 UI 交互模式。

## 二、聊天框翻译按钮 — 当前状态：不存在 ❌

### 当前翻译触发方式

聊天框中**没有翻译按钮**。翻译功能目前只能通过以下方式触发：

1. **控制台命令**：`translate [name]` / `translate_id <id>`
2. **BindChat 快捷键**：`!translate` / `!translateid`
3. **自动翻译**：当 `tc_translate_auto` 配置开启时自动触发

***

## 三、聊天行渲染机制

### 渲染流程

#### 1. `OnPrepareLines()` — 准备阶段

- 遍历 `m_aLines[]` 数组（最多64条，环形缓冲区）
- 计算每行的 Y 偏移量
- 使用 `TextRender()->CreateOrAppendTextContainer()` 创建文本容器
- 使用 `Graphics()->CreateRectQuadContainer()` 创建背景矩形
- 处理翻译文本的布局：翻译文本显示在上方（正常字号），原文在下方（0.8倍字号）

#### 2. `OnRender()` — 渲染阶段

- 渲染聊天输入框（当 `m_Mode != MODE_NONE` 时）
- 遍历所有可见行，应用动画效果：
  - **淡入/淡出**：`CalculateAnimationAlpha()` 基于 `MessageAge`
  - **滑入/滑出**：`CalculateAnimationOffsetX()`
  - **截断动画**：`CalculateCutOffAlpha()` / `CalculateCutOffOffsetX()`
- 渲染背景、Tee 头像、文本

### 每行渲染内容

每条聊天消息包含：

- **背景矩形**：圆角矩形
- **Tee 头像**：玩家角色小图标
- **好友标记**：`♥ `  前缀
- **客户端ID**：可选显示
- **玩家名**：带颜色
- **重复计数**：`[N]` 标记
- **消息文本**：带颜色
- **翻译文本**（如果有）：显示在原文上方，带语言标记 `[zh]`

### 关键坐标信息

- 聊天区域起始 X：`5.0f`
- 每行的 Y 偏移和文本位置存储在 `Line.m_TextYOffset`
- 行高存储在 `Line.m_aYOffset[OffsetType]`

***

## 四、聊天输入机制

聊天输入框仅包含：

- **模式标签**：显示 "All:" / "Team:" / "Chat:"
- **文本输入字段**：使用 `CLineInputBuffered<256>` 渲染
- **命令自动补全提示**：输入 `/` 时显示匹配的服务器命令
- **滚动支持**：多行输入时的平滑滚动

**没有任何额外按钮**（如发送按钮、翻译按钮、表情按钮等）。

***

## 五、添加聊天消息翻译按钮所需的工作

### 核心挑战

1. **聊天框没有鼠标交互基础设施**：需要从零开始添加鼠标追踪和点击检测
2. **聊天行使用预渲染容器**：文本和背景通过 `TextContainer`/`QuadContainer` 预渲染，按钮需要额外渲染
3. **聊天行位置是动态的**：由于动画效果（淡入/滑出/截断），行位置每帧都在变化

### 需要修改的文件

| 文件       | 修改内容                                  |
| -------- | ------------------------------------- |
| chat.h   | 添加鼠标状态追踪成员变量、悬停行索引、按钮ID               |
| chat.cpp | 添加 `OnCursorMove()` 覆写、悬停检测、按钮渲染和点击处理 |

### 具体实现步骤

#### 步骤1：添加鼠标追踪基础设施

在 `chat.h` 的 CChat 类中添加：

```cpp
int m_HoveredLine;           // 当前悬停的行索引（-1 = 无）
vec2 m_MousePos;             // 当前鼠标位置（屏幕坐标）
bool m_TranslateButtonHovered; // 翻译按钮是否被悬停
```

覆写 `OnCursorMove()` 以追踪鼠标位置。

#### 步骤2：在 OnRender() 中计算悬停行

在渲染循环中，利用已有的行位置信息检测鼠标是否在某行范围内：

```cpp
CUIRect LineRect = {x + AnimOffsetX, y, FullWidth, LineHeight};
vec2 MousePos = vec2(Ui()->MouseX(), Ui()->MouseY());
if(LineRect.Inside(MousePos))
    m_HoveredLine = i;
```

#### 步骤3：渲染翻译按钮

当行被悬停且该行没有翻译结果时，在行的右侧渲染一个翻译图标按钮。

#### 步骤4：处理按钮点击

在 `OnInput()` 或新增的鼠标事件处理中检测点击，调用 `GameClient()->m_Translate.Translate(Line)`。

#### 步骤5：处理已翻译行的交互

对于已有翻译的行，可以显示"隐藏翻译"或"重新翻译"按钮。

### 需要注意的问题

1. **坐标系统**：聊天渲染使用自定义坐标系（300.0f 高度），而 `Ui()->MouseX/Y()` 返回的可能是不同的坐标空间，需要做坐标转换
2. **动画偏移**：行位置受 `AnimOffsetX` 和 `AnimAlpha` 影响，悬停检测需要考虑这些偏移
3. **性能**：每帧检测所有可见行的悬停状态需要额外计算，但64行上限不会有性能问题
4. **旧版聊天模式**：`g_Config.m_ClChatOld` 模式下没有背景矩形，按钮渲染需要不同处理
5. **聊天输入模式冲突**：当聊天输入框激活时，鼠标点击不应触发翻译按钮
6. **CLine 生命周期**：CLine 使用环形缓冲区，行可能被覆盖，需要确保悬停索引有效
