# QmClient 钩子反馈时间进度条功能报告

## 一、功能概述

为钩子技能添加时间进度条，显示在钩子释放者角色旁边，准确反映钩子剩余持续时间。当玩家获得"无限钩"状态时，进度条替换为 ♾️ 符号显示。

- `HookDuration < 1.25f` → `m_HookTick` 为正数，钩子持续更短

## 三、当前问题

1. **钩子剩余时间不可见**：玩家无法直观感知钩子还有多少时间会断开，只能凭经验判断
2. **无限钩状态不够醒目**：当前只有 HUD 小图标，在激烈游戏中容易被忽略
3. **名字板没有钩子时间信息**：虽然已有钩子强弱图标，但没有时间维度信息

---

## 四、实现方案

### 4.1 钩子时间进度条

#### 4.1.1 数据获取

客户端已有所有必要数据：

| 数据 | 来源 | 位置 |
|------|------|------|
| `m_HookState` | `CNetObj_CharacterCore` | 每帧快照 |
| `m_HookTick` | `CNetObj_CharacterCore` | 每帧快照 |
| `m_EndlessHook` | `CNetObj_DDNetCharacter` 标志位 | 每帧快照 |
| `HookDuration` | `m_Tuning` | 调谐参数 |
| `SERVER_TICK_SPEED` | 常量 | 50 ticks/秒 |

#### 4.1.2 进度计算

```cpp
float GetHookProgress(const CNetObj_Character &Char)
{
    if(Char.m_HookState != HOOK_GRABBED)
        return 0.0f;

    if(Char.m_EndlessHook)
        return -1.0f;  // 特殊值：无限钩

    // 钩住玩家时：最大 1.2 秒
    // 钩住地面时：最大 HookDuration 秒
    float MaxTicks;
    if(Char.m_HookedPlayer != -1)
        MaxTicks = SERVER_TICK_SPEED * 1.2f;
    else
        MaxTicks = SERVER_TICK_SPEED * HookDuration;

    float ElapsedTicks = (float)Char.m_HookTick;
    float Progress = 1.0f - (ElapsedTicks / MaxTicks);
    return std::clamp(Progress, 0.0f, 1.0f);
}
```

#### 4.1.3 渲染位置

进度条显示在钩子释放者角色旁边，有两种方案：

**方案 A：名字板扩展（推荐）**

在名字板系统中新增 `CNamePlatePartHookTimer` 组件：

```
      [Strong] ← 已有钩子强弱图标
   ████████░░ ← 新增：钩子时间进度条
      PlayerName
      ClanName
```

- 位置：名字板中钩子强弱图标下方
- 大小：与名字板宽度一致，高度 2-3 像素
- 颜色：从绿→黄→红渐变（剩余时间从多到少）

**方案 B：Tee 旁边浮动条**

在 Tee 角色模型正上方渲染小型进度条：

```
    ██████░░
      🧑
     /||\
      /\
```

- 位置：Tee 头顶偏上
- 大小：Tee 宽度的 60%，高度 2 像素
- 优点：不依赖名字板开启，更直观
- 缺点：可能与名字板重叠

### 4.2 无限钩 ♾️ 符号显示

当 `m_EndlessHook == true` 且 `m_HookState == HOOK_GRABBED` 时：

**方案 A：进度条位置替换**

将进度条区域替换为 ♾️ 符号渲染：

```
      ♾️  ← 无限钩时显示
      PlayerName
```

**方案 B：钩头位置显示**

在钩链末端（钩头位置）显示 ♾️ 符号，更直观地表示"钩子无限"：

```
  PlayerName---♾️---🧑
```

### 4.3 配置变量

| 变量 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `qm_hook_timer` | INT | 0 | 钩子时间进度条（0=关, 1=仅自己, 2=所有人） |
| `qm_hook_timer_color` | COL | `65280` | 进度条颜色（绿色默认） |
| `qm_hook_timer_infinite_symbol` | INT | 1 | 无限钩时显示 ♾️ 符号 |

### 4.4 进度条视觉设计

```
剩余时间多：  ████████████████████  绿色
剩余时间中：  ██████████░░░░░░░░░░  黄色
剩余时间少：  ████░░░░░░░░░░░░░░░░  红色（闪烁）
无限钩：      ♾️                     特殊符号
```

颜色渐变规则：
- Progress > 0.6：绿色 `(0.3, 1.0, 0.3)`
- 0.3 < Progress ≤ 0.6：黄色 `(1.0, 0.9, 0.2)`
- Progress ≤ 0.3：红色 `(1.0, 0.3, 0.3)`，可加闪烁效果

## 五、实现路径

### 阶段一：名字板钩子进度条

1. 在 `CNamePlateData` 中添加 `m_HookProgress` 和 `m_HookEndless` 字段
2. 新增 `CNamePlatePartHookTimer` 名字板组件
3. 在名字板渲染流程中计算和渲染进度条

**修改文件**：

| 文件 | 修改内容 |
|------|----------|
| `nameplates.cpp` | 新增 `CNamePlatePartHookTimer` 类，计算进度，渲染进度条 |
| `nameplates.h` | 添加 `m_HookProgress`、`m_HookEndless` 数据字段 |
| `config_variables_qimeng.h` | 添加 `qm_hook_timer` 等配置变量 |

### 阶段二：无限钩 ♾️ 符号

1. 在进度条位置替换渲染 ♾️ 符号
2. 使用 Unicode 字符渲染或自定义精灵

### 阶段三：增强效果

1. 进度条颜色渐变（绿→黄→红）
2. 低时间闪烁效果
3. 钩子断开瞬间的视觉反馈（如进度条消失动画）
